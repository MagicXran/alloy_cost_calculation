# 4. 目标成分类型重新设计

你补充的低 Si 场景非常重要。

系统不能只支持：

```text

```

应支持多种成分目标类型。

---

## 4.1 目标类型一：范围控制 RANGE

适合常规元素。

```text
lower_limit ≤ Final_i ≤ upper_limit
```

例如：

```text
C: 0.140% ~ 0.180%
Mn: 1.150% ~ 1.250%
```

---

## 4.2 目标类型二：目标值加减余量 TARGET_BAND

适合你说的：

> 存在安全余量，要求在加减余量范围内。

例如：

```text
Mn 目标 = 1.20%
允许范围 ±0.03%
```

转成约束：

```text
1.17% ≤ Mn ≤ 1.23%
```

---

## 4.3 目标类型三：低于目标值，越低越好 UPPER_LOW_BETTER

适合低 Si、低 P、低 S 等需求。

例如：

```text
Si ≤ 0.040%
并且在成本可接受范围内，Si 越低越好
```

这个不能简单写成：

```text
Si ≤ 0.040%
```

因为你还希望：

```text
在最多多花 +1 元/t 的成本窗口内，Si 越低越好。
```

推荐求解方式：

```text
Step 1：求满足 Si ≤ 0.040% 的最低成本方案 C*
Step 2：增加约束 Cost ≤ C* + 1 元/t
Step 3：在这个成本窗口内，Minimize Final_Si
Step 4：把多个方案都列出来，让用户决策
```

---

## 4.4 低 Si 多方案输出

对于低 Si，系统应该至少输出这些方案：

| 方案 | 求解目标 | 说明 |
|---|---|---|
| A 成本最低方案 | 满足 Si ≤ 目标，成本最低 | 作为成本基线 |
| B +0.5 元/t 低 Si 方案 | Cost ≤ C* + 0.5，Si 最低 | 小幅加钱换更低 Si |
| C +1.0 元/t 低 Si 方案 | Cost ≤ C* + 1.0，Si 最低 | 你指定的推荐窗口 |
| D 鲁棒低 Si 方案 | 考虑回收率/成分波动后仍满足低 Si | 风险更低 |
| E 禁用硅锰/硅铁方案 | 现场规则强制禁用硅源合金 | 特殊现场规则方案 |

| 方案 | 成本 元/t | 成本增量 | 预测 Si | 使用合金 | 风险 |

# 5. 安全余量与成分波动范围的关系

你之前问过这个点，这里结合新版方案再明确一次。

二者不是一回事。

## 5.1 安全余量

安全余量是管理控制策略。

例如：

```text
C 内控范围：0.140% ~ 0.180%
安全余量：上限 0.005%
```

则优化时用：

```text
C ≤ 0.175%
```

本质是：

```text
我不希望方案贴着边界。
```

---

## 5.2 成分波动范围

成分波动范围是参数不确定性。

例如：

```text
终点 C 名义值 = 0.060%
终点 C 高值 = 0.065%

高碳锰铁 C 名义含量 = 7.0%
高值 = 7.5%

C 回收率名义值 = 0.95
高值 = 0.98
```

本质是：

```text
输入参数本身不准，要考虑最差情况。
```

---

## 5.3 两者在模型里的组合

普通模型：

```text
Final_C ≤ C_upper
```

加安全余量：

```text
Final_C ≤ C_upper - upper_margin
```

加鲁棒波动：

```text
WorstHigh_C ≤ C_upper - upper_margin
```

其中：

```text
WorstHigh_C =
终点C高值 + Σ(合金加入量 × 合金C高值 × C高回收率 / 10)
```

所以：

```text
安全余量 = 收缩目标边界
成分波动 = 改变成分预测的保守计算方式
```

二者可以同时存在，但不能盲目都设很大，否则方案会过度保守、成本明显升高。

# 7. 炉重设计：默认不强关联炉重

你的判断是对的。

这个系统首先应该按：

```text
kg/t钢
元/t钢
```

来做。

炉重应作为可选参数。

---

## 7.1 三种模式

| 模式 | 是否需要炉重 | 是否包装取整 | 适用场景 |
|---|---|---|---|
| 吨钢理论模式 | 不需要 | 不启用 | 成本测算、销售评估、效益分析 |
| 参考炉重模式 | 可选 | 可不启用 | 工艺人员看 kg/炉 |
| 生产投料模式 | 需要 | 启用 | 现场实际投料 |

---

## 7.2 最小投料量也应按模式启用

你对“最小投料量”的理解是对的。

它不是说候选合金必须投，而是：

> 如果优化器选择了某个合金，则这个合金不能低于现场可执行的最小投料量。

数学上：

```text
x_j ≤ max_j × y_j
x_j ≥ min_j × y_j
```

如果：

```text
y_j = 0
```

则：

```text
x_j = 0
```

如果：

```text
y_j = 1
```

则：

```text
min_j ≤ x_j ≤ max_j
```

在不输入炉重的吨钢理论模式下，建议默认不启用“每炉最小投料 kg”约束，否则没有实际意义。

# 8. 规则层：必须增加，而且要产品化

你新增的规则层是整个系统能不能通用落地的关键。

不同现场一定会有特殊规则，例如：

```text
当 Si目标 <= 0.040 时，禁用硅锰/硅铁；
低碳钢禁用高碳锰铁；
Mn 缺口允许金属锰兜底；
某现场优先使用库存充足的合金；
某钢种不允许使用含硼合金；
某客户订单禁用某供应商合金。
```

这些不能全部写死在代码里。

# 9. 规则层的正确设计：规则包 + 规则引擎 + 约束注入

建议系统设计成：

```text
通用优化内核
    +
可插拔规则包
```

也就是：

```text
不同钢厂、不同产线、不同现场，可以挂不同规则包。
```

---

## 9.1 规则层应该干什么？

规则层不是单纯判断提示，它要参与模型构建。

它可以做五类事情：

| 规则类型 | 作用 | 示例 |
|---|---|---|
| 候选合金过滤 | 禁用或启用某些合金 | 低 Si 时禁用硅锰/硅铁 |
| 参数覆盖 | 修改回收率、安全余量、目标值 | 某现场 Ti 回收率降低 |
| 约束注入 | 加入额外数学约束 | 高碳合金总量 ≤ 某值 |
| 目标调整 | 给某些合金加惩罚或偏好 | 金属锰作为兜底，不优先用 |
| 后处理解释 | 生成业务解释和风险提示 | 因 Si 目标低，已禁用硅源合金 |

---

## 9.2 规则执行位置

规则不应该只在求解后执行，而应该贯穿建模过程。

推荐流程：

```text
输入场景
    ↓
读取炼钢牌号基础目标
    ↓
读取订单/用途特殊要求
    ↓
读取候选合金
    ↓
执行规则层：候选过滤、参数覆盖、约束注入
    ↓
生成 MILP / 鲁棒 MILP 模型
    ↓
求解
    ↓
执行后处理规则：解释、警告、替代建议
```

# 12. 你举的两个规则如何建模？

## 12.1 规则一：Si 目标 ≤ 0.040 时，禁用硅锰/硅铁

业务规则：

```text
当 Si目标 <= 0.040 时，直接禁用硅锰/硅铁两个变量。
```

模型动作：

```text
y_硅锰 = 0
y_硅铁 = 0
```

或者直接从候选合金中移除。

规则定义：

```text
条件：
    target("Si").upper <= 0.040

动作：
    DISABLE_ALLOY ["硅锰", "硅铁"]

解释：
    当前 Si 控制目标不高于 0.040%，为避免硅源合金带入 Si，系统禁用硅锰和硅铁。
```

需要注意：

> 如果禁用硅锰，Mn 的来源就会受限，所以必须允许低碳锰铁、金属锰或其他锰源兜底，否则模型可能无解。

---

## 12.2 规则二：Mn 允许金属锰作为兜底

这里有两种实现方式。

### 方式 A：金属锰始终可用，但成本自然较高

如果金属锰价格明显高，优化器只有在必要时才会使用。

优点：

```text
简单
求解稳定
不需要特殊逻辑
```

缺点：

```text
如果价格数据不合理，可能被过度使用。
```

---

### 方式 B：金属锰作为兜底料，带惩罚项

目标函数变成：

```text
Minimize 合金成本 + 金属锰兜底惩罚
```

例如：

```text
真实成本 = 20000 元/t
兜底惩罚 = 2 元/t钢
```

这样模型会更倾向先用普通锰源，只有必要时才用金属锰。

---

### 方式 C：先求普通方案，不可行再启用金属锰

求解流程：

```text
Step 1：不启用金属锰求解
Step 2：如果无解，再启用金属锰重新求解
Step 3：标记为“兜底方案”
```

这个方式业务解释最好。

系统可以提示：

```text
常规合金组合无法同时满足低 Si 与 Mn 下限；
已启用金属锰作为兜底料，成本增加 2.6 元/t。
```

我建议正式产品采用：

```text
方式 B + 方式 C
```

即：

```text
金属锰默认作为兜底候选；
常规方案优先；
若常规方案无解或风险高，再生成金属锰兜底方案。
```

---

# 13. 规则冲突必须处理

规则层灵活之后，最大风险是规则互相打架。

例如：

```text
规则 A：低 Si 禁用硅锰
规则 B：某现场优先使用硅锰
```

或者：

```text
规则 A：禁用金属锰
规则 B：Mn 缺口允许金属锰兜底
```

必须设计冲突处理。

---

## 13.1 规则优先级

建议按优先级排序：

```text
订单特殊要求
    >
客户要求
    >
炼钢牌号规则
    >
现场规则
    >
全局规则
```

或者：

```text
安全/质量硬规则
    >
工艺硬规则
    >
成本偏好规则
    >
推荐/解释规则
```

---

## 13.2 硬规则与软规则

| 类型 | 含义 |
|---|---|
| HARD | 必须执行，违反则方案不可用 |
| SOFT | 偏好规则，可被成本或可行性覆盖 |

例如：

```text
低 Si 禁用硅锰
```

如果是客户明确要求，应该是 HARD。

```text
优先使用硅锰
```

通常是 SOFT。

---

## 13.3 规则冲突时不能静默覆盖

系统必须提示：

```text
规则冲突：
R001 低 Si 禁用硅锰；
R017 现场优先使用硅锰；
已按 HARD 规则执行，硅锰被禁用。
```

# 18. 一条完整业务流示例：低 Si 订单

销售输入：

```text
成品钢种：Q235B-WJ
厚度：1.5 mm
宽度：1250 mm
特殊要求：Si ≤ 0.040%
成本窗口：+1 元/t
```

系统流程：

```text
1. Q235B-WJ 映射到炼钢牌号 L001
2. 读取 L001 的唯一内控目标
3. 叠加订单特殊要求：Si ≤ 0.040，且低 Si 优先
4. 触发规则：
   target("Si").upper <= 0.040
   → 禁用硅锰、硅铁
   → 启用金属锰兜底
5. 求解成本最低方案 C*
6. 在 Cost ≤ C* + 1 元/t 下求 Si 最低方案
7. 同时生成金属锰兜底方案、鲁棒低 Si 方案
8. 输出多方案
9. 查询该炼钢牌号在 1.5mm × 1250mm 分档下历史效益
10. 给销售展示辅助报价风险
```

输出示例：

| 方案 | 成本 元/t | 成本增量 | Si | 主要合金 | 风险 |
|---|---:|---:|---:|---|---|
| 成本最低 | 84.2 | 0 | 0.039 | 低碳锰铁 + 少量硅源 | 中 |
| 规则禁硅源 | 86.7 | +2.5 | 0.018 | 金属锰 + 低碳锰铁 | 低 |
| +1窗口低Si | 85.2 | +1.0 | 0.028 | 金属锰少量替代 | 中低 |
| 鲁棒低Si | 86.1 | +1.9 | 0.026 | 金属锰 + 低碳锰铁 | 低 |

销售看到后，可以判断：

```text
如果客户只要求 Si ≤ 0.040，成本最低方案可能够用；
如果客户强烈偏好低 Si，建议采用 +1 元/t 低 Si 方案；
如果要求严格低 Si 且风险低，需要报价覆盖 +1.9~2.5 元/t 成本增量。
```

例如：

```text

```

这条规则如果是硬规则，系统必须强制：

# 热卷合金成本最优计算工具 — Spec & 实施计划

## 1. Context & Objective

### 问题本质
转炉/精炼工序向钢水加入多种铁合金（硅锰、高/中/低碳锰铁、硅铁、高/低碳铬铁），满足钢种目标成分（C/Si/Mn/Cr），同时最小化吨钢合金成本。每种合金同时带入多种元素（尤其碳），构成**多元素耦合约束下的成本最小化问题**。

### 工具定位
**离线桌面计算工具** — 不接入MES/LIMS/Oracle等外部系统，手工录入参数，本地求解输出。面向炼钢工艺工程师日常使用。

### 参考资料综合判断

| 来源 | 采纳什么 | 不采纳什么 |
|------|----------|------------|
| PPT原始公式 | 7种合金数据、碳预算概念、加入顺序经验 | 嵌套IF-ELSE贪心逻辑（🔴垃圾） |
| 流程图(流程.png) | BOF+LF两阶段概念（简化为手动多轮迭代） | Oracle集成、K-means/PCA/BP在线预测 |
| 界面图(页面参考.png) | UI分区布局思路、合金启用/禁用勾选、成分上下限显示 | 在线数据库查询功能 |
| ChatGPT分析 | MILP公式、安全余量、回收率per-alloy×element、温降估算、历史对比 | 多阶段自动闭环、FastAPI微服务、动态回收率预测引擎 |
| 参考2.md | 三级方案（规则→LP→MILP）、温度惩罚概念、P/S杂质硬约束、批量整数约束 | PCA-DNN/t-SNE-WOA-LSTM、Gurobi商业求解器、L2实时通讯 |

### PPT方案致命缺陷（不改不行）

1. **贪心算法不保证全局最优** — 先吃完碳预算给铬铁，导致后面补锰被迫用贵的低碳锰铁
2. **魔法阈值(0.05/0.03/0.02)无数学依据** — 应由LP自动处理碳约束
3. **高锰修正(-0.08%)是补丁** — 数据结构设计错误的症状
4. **回收率处理前后矛盾** — 有时用有时不用
5. **输出浮点数脱离现场** — 现场按整袋/整批投料

## 2. 数学模型（第一性原理推导）

### 2.1 元素质量平衡方程

对1吨钢水（W=1000kg），加入 x_j kg 的合金 j，元素 e 的成分增量：

```
Δ_e(%) = x_j × a_je(%) / 100 × η_je / W(kg) × 100
       = x_j × a_je × η_je / 1000
```

**验证**：硅锰10kg/t，Mn=65.66%，η=0.98 → 10×65.66×0.98/1000 = **0.643%** ✓

> ⚠️ 注意：因子是 **/1000** 不是 /10000。推导：(÷100 %转小数) × (×100 小数转%) × (÷1000 per-ton) = 1/1000

### 2.1b 炉次重量与单位体系

引入 `heat_weight_t`（实际炉次钢水量，如132.2t）。计算体系以 **kg/t** 为内部单位，输出时同时给出：

| 输出 | 公式 | 单位 |
|------|------|------|
| 单耗 | x_j | kg/t |
| 炉次加入量 | x_j × heat_weight_t | kg/炉 |
| 袋数/批数 | x_j × heat_weight_t / bag_size_kg（取整） | 袋 |
| 吨钢成本 | Σ P_j × x_j | ¥/t |
| 炉次合金成本 | 吨钢成本 × heat_weight_t | ¥/炉 |

MILP整袋约束正确形式：`x_j × heat_weight_t = n_j × bag_size_kg`，即步长 = `bag_size_kg / heat_weight_t`

### 2.2 优化模型标准形式

**决策变量**：x_j = 合金 j 加入量 (kg/t钢), j=1..n

**目标函数**：
```
Minimize Z = Σ_j (P_j × x_j)    [¥/t钢, P_j单位为¥/kg]
```
> PPT价格单位是¥/t合金，需转换：P_j(¥/kg) = PPT价格 / 1000

**约束条件**：

| 约束类型 | 公式 | 说明 |
|----------|------|------|
| 元素下限 | Residual_e + Σ_j(x_j·a_je·η_je/1000) ≥ Target_e_min | 成分不能低于目标下限 |
| 元素上限 | Residual_e + Σ_j(x_j·a_je·η_je/1000) ≤ Target_e_max | 成分不能超过目标上限 |
| P/S杂质 | Residual_P + Σ_j(x_j·P_j_content·η_P/1000) ≤ P_max | 杂质不可逾越红线 |
| 非负 | x_j ≥ 0 | 合金量不能为负 |
| 整数批量(MILP) | x_j × heat_weight_t = n_j × B_j, n_j∈Z≥0 | 按整袋投料，步长=B_j/heat_weight_t |
| 单耗上限 | x_j ≤ max_add_kg_per_t | 防止异常解（可选） |

### 2.3 回收率模型（三级设计）

| 级别 | 方式 | V1实现 |
|------|------|--------|
| 第一级 | 全局per-element默认值（PPT方式：C=0.9, Si=0.75, Mn=0.98, Cr=0.95） | ✅ |
| 第二级 | per-alloy×element矩阵（如硅锰的Mn回收率≠高碳锰铁的Mn回收率） | ✅ config中可选覆盖 |
| 第三级 | ML动态预测（K-means/PCA/DNN） | ❌ V2再考虑 |

### 2.4 安全余量机制（双边设计）

每个元素独立配置上下两侧安全余量，不再按"有益/有害"粗分：

```
有效下限 = Target_min + safety_low
有效上限 = Target_max - safety_high
```

配置格式：
```json
"safety_margins": {
  "C":  {"low": 0.000, "high": 0.005},
  "Si": {"low": 0.010, "high": 0.010},
  "Mn": {"low": 0.010, "high": 0.010},
  "Cr": {"low": 0.005, "high": 0.005},
  "P":  {"low": 0.000, "high": 0.002},
  "S":  {"low": 0.000, "high": 0.002}
}
```

为什么双边：Mn虽然是目标元素但也不能超上限；C在某些钢种也有下限。统一公式消除特殊分支。

### 2.5 温度降估算（V1默认禁用）

V1**不输出温降数值**——经验公式精度不足（6°C/t合金的系数有争议，典型场景下结果偏低两个数量级）。

V1只输出：`总合金加入量: xx kg/t`。温降模型保留为配置项，默认关闭：
```json
"temperature_drop": {"enabled": false}
```
V2引入per-alloy降温系数并经过现场校验后再启用。

## 3. 三种求解模式

参考参考2.md的分层方案，工具内置三种模式，同时运行、对比输出：

| 模式 | 算法 | 输出特征 | 用途 |
|------|------|----------|------|
| **A: 规则基线** | PPT的IF-ELSE启发式 | 连续浮点，局部最优 | 对照基线，证明LP优势 |
| **B: LP连续最优** | scipy linprog (HiGHS) | 连续浮点，全局最优 | 理论最低成本参考 |
| **C: MILP实用最优** | scipy milp (HiGHS) | 整数批量，全局最优 | 现场可执行方案 |

三个方案同时输出，用户一眼看出差距。

## 7. 核心算法实现要点

### 7.0 配置校验器（load_config时自动运行）

加载config后立即进行合理性检查，发现问题直接报错中止：

| 字段 | 合理范围 | 疑似单位错误提示 |
|------|----------|------------------|
| 合金成分(%) | 0~100 | <1时提示"是否应为百分比？如0.6566应为65.66" |
| 回收率 | 0~1.2 | >1.2时提示"回收率异常" |
| 目标成分(%) | 0~10 | >10时提示"是否单位错误" |
| 价格(¥/t) | 100~200000 | <100时提示"是否应为¥/t而非¥/kg" |
| bag_size(kg) | 0~2000 | — |
| heat_weight_t | 1~500 | — |
| 安全余量 | 0~0.5 | >0.1时提示"安全余量过大" |

### 7.0b P/S超标预检查

求解前检查：
1. `residual_P > target_P_max` → 报错："当前P=x%已超过目标上限y%，加合金无法降磷，需工艺处理（吹氧去磷等）"，终止优化
2. `residual_S > target_S_max` → 同理
3. 合金P/S成分使用默认值（PPT未给出）时 → 输出黄色警告："P/S含量为行业默认值，非实际化验数据，建议用供应商质保书或化验单覆盖"

### 7.1 LP求解 (Mode B)

```python
from scipy.optimize import linprog

# 目标：min c^T x
c = [alloy.price_per_ton / 1000 for alloy in enabled_alloys]  # ¥/kg

# 约束：A_ub @ x <= b_ub
# 对每个有上限的元素e:  Σ_j(x_j * a_je * η_je / 1000) <= Target_max - Residual - safety_high
# 对每个有下限的元素e: -Σ_j(x_j * a_je * η_je / 1000) <= -(Target_min - Residual + safety_low)
# 每种合金上限: x_j <= max_add_kg_per_t

result = linprog(c, A_ub=A_ub, b_ub=b_ub, bounds=bounds, method='highs')
```

### 7.2 MILP求解 (Mode C)

```python
from scipy.optimize import milp, LinearConstraint, Bounds

# 对有bag_size的合金：步长 = bag_size_kg / heat_weight_t
# 决策变量改为 n_j（整数袋数），x_j = n_j × step_j
# scipy.optimize.milp 支持 integrality 参数
# 约束矩阵中系数相应调整为 step_j × a_je × η_je / 1000
```

### 7.2b 无可行解诊断

当LP或MILP返回infeasible时，自动运行诊断模型：

```python
# 给每个元素约束加松弛变量 s_e+（超上限量）和 s_e-（差下限量）
# 目标：min Σ_e M_e × (s_e+ + s_e-)
# 该模型一定有可行解
# s_e > 0 的约束即为冲突源
```

输出示例：
```
⚠️ 无可行解！约束冲突诊断：
  Mn下限: 差 0.035%（需1.100%，当前最多到1.065%）
  C上限:  超 0.006%（限0.100%，当前最少到0.106%）
建议：放宽Mn下限 或 启用低碳锰铁 或 放宽C上限
```

### 7.3 PPT规则基线 (Mode A)

按PPT顺序实现嵌套IF-ELSE逻辑，作为对比基线：
1. 高碳铬铁（碳预算>0.05则加）
2. 低碳铬铁（补足剩余铬）
3. 硅锰（min(硅需求, 锰需求)决定量）
4. 高碳锰铁+硅铁（碳允许则用高碳锰铁补锰，硅铁补硅）
5. 中碳锰铁（碳余量>0.02）
6. 低碳锰铁（兜底补锰）

### 7.4 结果后处理与警告

- **成分贴边警告**：如果某元素最终成分距边界 < safety_margin，输出⚠️
- **LP极小量标注**：如果某合金LP结果 < 1 kg/t 且 bag_size=0，标注"极小量，建议忽略"
- **P/S默认值标记**：使用默认P/S数据计算的结果，标注"P/S为默认值"

## 9. 多轮迭代支持（替代自动闭环）

虽然不做BOF→LF自动闭环，但支持**手动多轮计算**：

1. 第一轮：用终点样成分作为residual → 得到出钢阶段加料方案
2. 第二轮：LF取样后，手动更新config中residual为最新成分 → 重新运行 → 得到补加方案
3. 第三轮：再次取样，再次运行...直到合格

这完全模拟了流程图(流程.png)的BOF→LF多阶段逻辑，只是数据输入是手动的。

### 不采纳的理由总结

**回收率三值(mean/low/high)**：与双边安全余量功能重叠。安全余量是固定偏移，已经补偿了回收率波动。同时用两套保守机制 = 双重保守 = 方案偏贵。且工程师未必能给出回收率区间数据。

**Excel输入**：增加openpyxl依赖，违背"零额外依赖"原则。工艺工程师编辑JSON确实不友好，但V1配置校验器可以拦截大部分错误。V2做Excel模板。

**最大合金种类数约束**：LP/MILP的稀疏解特性已经自然趋向少种类使用。人为限制种类数反而可能排除最优解。

**min_add_kg_per_heat、allowed_stages、inventory**：V1单阶段离线工具不跟踪库存不区分阶段。MILP整袋约束已经消除了极小量问题（bag_size>0的合金，结果要么是0要么是整袋倍数）。

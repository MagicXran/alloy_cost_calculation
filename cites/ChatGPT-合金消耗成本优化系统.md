# 1. 对第一张计算流程图的识别与判断

第一张图的核心流程可以拆成两条线。

## 1.1 左侧是生产执行主流程

它大致是：

1. 读取炉号；
2. 匹配钢种；
3. 从 Oracle 数据库读取数据；
4. 获取钢水信息、合金成分、价格、目标成分；
5. 模型计算合金配料方案；
6. 按计算结果加合金；
7. LF 阶段再次获取钢水与合金信息；
8. 重新设置模型参数并再次求解；
9. 测温、取样；
10. 判断成分是否达标；
11. 不达标则返回重新计算；
12. 达标后输出数据并反馈数据库。

这说明它不是“一次性计算”，而是典型的**BOF 出钢阶段粗配 + LF 精炼阶段微调 + 取样闭环**。

这点很重要。我们的工具不能只做“输入一次，输出一次”，而应该支持：

| 阶段 | 目标 |
|---|---|
| BOF / 出钢阶段 | 利用钢水搅拌条件，尽量使用便宜的大宗合金，快速接近目标下限 |
| LF 初期 | 根据最新样品，重新优化补加 |
| LF 中后期 | 小批量精调，避免超上限 |
| 取样后 | 判断是否合格，不合格则二次优化 |
| 完成后 | 记录实际命中率、成本、回收率，用于模型校正 |

AIST/SSAB 的合金加入模型也是类似思路：模型与液态钢生产单元实时交互，覆盖 EAF、LMF、VTD、Caster 等模块，并使用线性规划自动推荐合金种类和加入量，以保证质量和成本有效性。([aist](https://www.aist.org/AIST/aist/AIST/Publications/Digital%20Transformations/20-jan-digital-transformation.pdf))

---

## 1.2 右侧是智能回收率与经验学习流程

右侧流程包含：

- K-means 聚类；
- 钢种历史生产数据；
- 智能选择合金种类；
- 生产过程参数；
- 合金收得率；
- 主成分分析 PCA；
- BP 神经网络；
- 预测合金收得率；
- 合金信息数据库反馈。

这说明它想解决一个关键问题：

> 合金加入量不是单纯由理论成分决定，真正影响计算准确性的核心是“回收率/收得率”。

这个方向是正确的。文献中也有类似做法：针对 BOF/LF 铁合金消耗优化，有研究采用 K-means 聚类识别相似生产情景，再结合可解释模型给出铁合金消耗推荐。([arXiv](https://arxiv.org/abs/2204.07421))

但是我建议对这套思路做一次“工程化降维”。

**K-means + PCA + BP 神经网络可以参考，但不要一开始就作为主算法。**

原因是：

| 方法 | 价值 | 风险 |
|---|---|---|
| K-means | 找相似炉次，适合估算回收率 | 聚类边界不稳定，现场解释难 |
| PCA | 找主要影响因素，减少变量 | 对操作人员解释性一般 |
| BP 神经网络 | 可拟合非线性回收率 | 黑箱、样本量要求高、上线风险大 |
| 线性规划/MILP | 成本最低求解，逻辑清晰 | 依赖回收率准确性 |
| 规则引擎 | 符合现场经验 | 不一定全局最优 |

所以最稳妥方案是：

> **优化模型负责“算最低成本”，机器学习只负责“预测回收率/安全余量”。**

不要让神经网络直接决定加多少合金。神经网络可以预测“这炉硅锰 Mn 回收率大概是 0.96 还是 0.92”，但最终合金加入量仍由线性规划/整数规划求解。

## 2.2 合金回收率设置

界面中有 Mn 回收率、Cr 回收率，并按不同合金区分，例如高锰、中锰、硅锰、高铬、低铬等。

这比 PPT 更合理。PPT 里回收率按元素给出，最终页中 C、Si、Mn、Cr 分别是 0.9、0.75、0.98、0.95。`afa82181-5107-454d-a1d6-0df0ba9c79f1.pptx`  
但现场更准确的方式应该是：

> **回收率 = 元素 × 合金类型 × 加入阶段 × 钢种 × 工况 的函数。**

例如：

| 元素 | 合金 | 阶段 | 回收率应不同的原因 |
|---|---|---|---|
| Mn | 硅锰 | 出钢 | 出钢搅拌强，回收率较高 |
| Mn | 中碳锰铁 | LF | 加料条件不同，回收率可能不同 |
| Si | 硅锰/硅铁 | 出钢 | 易受氧化渣、钢水氧含量影响 |
| Cr | 高碳铬铁/低碳铬铁 | LF | 熔化速度、粒度、温度影响明显 |
| Al | 铝块/铝线 | LF | 受氧含量、渣氧化性影响很大 |

AIST/SSAB 的合金数据库也明确维护合金成本、化学成分、适用工位和不同元素回收率，回收率还可以根据历史数据动态调整。([aist](https://www.aist.org/AIST/aist/AIST/Publications/Digital%20Transformations/20-jan-digital-transformation.pdf))

| 限制类型 | 示例 |

所以我们的系统应支持：

# 3. 对我们原方案的关键优化

之前我们的方案是：

> 输入成分、合金品位、价格、回收率 → 用 LP/MIP 求最低成本。

现在结合这两张图，应该升级为：

> **炉次数据自动读取 → 动态回收率预测 → 多阶段合金优化 → 现场批量取整 → 成分闭环校正 → 历史反馈学习。**

核心架构如下。

## 3.1 第一层：数据采集层

来源包括：

| 数据 | 来源 |
|---|---|
| 炉号、钢种、出钢量 | MES / 二级系统 |
| 终点 C/Si/Mn/Cr/P/S | LIMS / 快分仪 |
| 出钢温度、氧含量 | 过程检测 |
| 合金成分 | 合金品质数据库 |
| 合金价格 | ERP / 采购 |
| 合金库存 | 仓储系统 |
| 目标成分上下限 | 钢种标准库 |
| 历史加料与命中数据 | 历史生产数据库 |

第一版可以手工录入，第二版再接 Oracle/MES/LIMS。

---

## 3.2 第二层：回收率模型层

建议分三级，不要一步到 BP 神经网络。

### 第一级：规则默认回收率

例如：

| 元素 | 默认回收率 |
|---|---:|
| C | 0.90 |
| Si | 0.75 |
| Mn | 0.96-0.98 |
| Cr | 0.95-0.96 |

这些可以从现有 PPT 和现场经验开始。`afa82181-5107-454d-a1d6-0df0ba9c79f1.pptx`

### 第二级：相似炉次修正

按照以下特征找相似炉次：

| 特征 | 说明 |
|---|---|
| 钢种 | 同钢种或同族钢种 |
| 出钢量 | 影响 kg/t 计算 |
| 终点 C/O | 影响氧化性和收得率 |
| 出钢温度 | 影响合金熔化和反应 |
| 终点 Mn/Si | 影响补加空间 |
| 渣况 | FeO、MnO、碱度 |
| 加料阶段 | BOF、LF、RH |
| 合金种类 | 硅锰、高碳锰铁、低碳锰铁等 |
| 加料量 | 小加料和大加料回收率可能不同 |

K-means 可以用于这个层面：先找“相似工况组”，再用该组的历史平均回收率修正当前炉次。

### 第三级：机器学习预测回收率

图中用的是 PCA + BP 神经网络。这个可以保留为高级模块，但我建议初期优先使用：

| 模型 | 建议 |
|---|---|
| 线性回归/岭回归 | 第一版推荐，解释性强 |
| 决策树/随机森林 | 第二版推荐，能捕捉非线性 |
| XGBoost/LightGBM | 数据量足够后推荐 |
| BP 神经网络 | 可作为对照模型，不建议第一版主用 |

原因是现场更需要“为什么这炉回收率低”的解释能力，而不是单纯黑箱预测。

## 3.3 第三层：优化求解层

这是系统的核心。

模型不再用 PPT 里的多层 IF，而是用统一约束求解。

\min Z=\sum_{j\in J}\frac{P_j}{1000}x_j+\lambda_1R+\lambda_2N+\lambda_3D\\
\text{s.t.}\quad C_e^{out}=C_e^{in}+\sum_{j\in J}\frac{x_j a_{j,e}\eta_{j,e}}{1000}+\Delta_e^{process}\\
L_e+m_e^-\le C_e^{out}\le U_e-m_e^+\\
x_j\ge0,\quad x_j W=q_j n_j,\quad n_j\in\mathbb{Z}_{\ge0}

其中：

| 符号 | 含义 |
|---|---|
| $x_j$ | 第 j 种合金单耗，kg/t |
| $P_j$ | 第 j 种合金价格，元/t |
| $a_{j,e}$ | 合金 j 中元素 e 的品位，% |
| $\eta_{j,e}$ | 合金 j 中元素 e 的预测回收率 |
| $C_e^{in}$ | 加料前钢液元素 e 成分，% |
| $C_e^{out}$ | 加料后预测元素 e 成分，% |
| $L_e, U_e$ | 目标成分下限、上限 |
| $m_e^-, m_e^+$ | 安全余量 |
| $W$ | 钢水量，t |
| $q_j$ | 合金最小加料批量，kg/批 |
| $n_j$ | 加料批数，整数 |

其中 $R$、$N$、$D$ 是现场化后的附加目标：

| 项 | 含义 |
|---|---|
| $R$ | 成分风险惩罚，避免贴边 |
| $N$ | 合金种类数量惩罚，避免方案过碎 |
| $D$ | 偏离目标值惩罚，避免只压下限 |

AIST/SSAB 的案例也明确把合金加入问题表述为成本最小化线性规划：目标是最小化合金加入成本，约束是加合金后的每个元素浓度满足钢种规格。([aist](https://www.aist.org/AIST/aist/AIST/Publications/Digital%20Transformations/20-jan-digital-transformation.pdf))

# 4. 最关键的现场化公式

## 4.1 成分预测公式

如果 $x_j$ 用 kg/t 表示，则元素 e 的增量为：

$$
\Delta C_e = \sum_j \frac{x_j \times a_{j,e} \times \eta_{j,e}}{1000}
$$

例如硅锰中 Mn = 65.66%，回收率 0.98，加入 10 kg/t，则 Mn 增量约为：

$$
10 \times 65.66 \times 0.98 / 1000 = 0.643\%
$$

最终预测成分：

$$
C_e^{out}=C_e^{in}+\Delta C_e+\Delta_e^{process}
$$

这里必须加入 $\Delta_e^{process}$，因为现场存在过程漂移：

| 元素 | 过程漂移示例 |
|---|---|
| C | 增碳、脱碳、包衬/覆盖剂影响 |
| Si | 氧化损失、渣带入影响 |
| Mn | 氧化损失、渣中 MnO 影响 |
| Cr | 熔化不足、氧化损失 |
| Al | 脱氧消耗剧烈 |
| S | 精炼过程脱硫，不是简单加和 |
| P | 基本不能靠合金调低，只能限制带入 |

---

## 4.2 成本最低公式

$$
\min \sum_j x_j \times \frac{P_j}{1000}
$$

其中 $P_j$ 是元/t 合金，$x_j$ 是 kg/t 钢。

如果现场价格是元/kg，则公式改成：

$$
\min \sum_j x_j \times P_j
$$

所以系统必须强制统一价格单位。第二张图界面里写的是“元/kg”，PPT 里是类似“元/t”的价格口径，这一点需要在工具中严格区分。否则成本会差 1000 倍。

---

## 4.3 碳约束公式

碳是整个模型最重要的硬约束。

$$
C^{in}+\sum_j \frac{x_j a_{j,C}\eta_{j,C}}{1000}+C^{pickup}\le U_C-m_C^+
$$

现场建议：

| 钢种类型 | 碳安全余量 |
|---|---:|
| 低碳钢 | 0.003-0.008% |
| 普碳钢 | 0.005-0.010% |
| 碳窗口很宽的钢种 | 可适当放宽 |
| 首次上线模型 | 建议偏保守 |

这比 PPT 中固定写死“增碳余量 0.03、0.05、0.02”的方式更稳。PPT 的阈值可以作为初始经验，但不应作为最终公式。`afa82181-5107-454d-a1d6-0df0ba9c79f1.pptx`

---

## 4.4 回收率稳健约束

现场不能只用一个平均回收率。建议用“保守回收率”进入优化：

对需要达到下限的有益元素，例如 Mn、Cr、Si：

$$
\eta_{used}=\hat{\eta}-k\sigma
$$

对容易超上限或有害元素，例如 C、P、S、B：

$$
\eta_{used}=\hat{\eta}+k\sigma
$$

这样做的含义是：

| 元素类型 | 优化时采用 |
|---|---|
| 需要补足的元素 | 假设回收率偏低，避免补不够 |
| 不能超标的元素 | 假设带入偏高，避免超标 |
| 成本计算 | 用实际加入量计算 |
| 预测展示 | 同时展示平均值、保守值、风险区间 |

这比单点回收率更贴合现场。

# 5. 我建议优化后的完整系统流程

## 5.1 BOF 出钢阶段

目标：  
大宗合金低成本粗配，优先接近下限，不追求一次到目标中心。

逻辑：

1. 读取炉号、钢种、出钢量、终点样；
2. 读取钢种目标下限/上限；
3. 读取允许 BOF 使用的合金；
4. 调用 BOF 阶段回收率；
5. 求解最低成本方案；
6. 输出加料方案；
7. 记录实际加入。

此阶段可以允许更多使用便宜合金，例如硅锰、高碳锰铁、高碳铬铁，但必须受 C 上限约束。

---

## 5.2 LF 精炼阶段

目标：  
根据新样品进行精准补加，控制成分命中率。

逻辑：

1. 读取 LF 样品；
2. 对比目标范围；
3. 如果全部合格，只输出“无需补加”；
4. 如果部分元素低于下限，执行补加优化；
5. 如果部分元素接近上限，自动禁用高风险合金；
6. 输出小批量加料方案；
7. 测温取样后再次判断。

AIST/SSAB 的案例也提到，LMF 和 VTD 阶段通常需要多次化验，优化程序应在收到当前化验数据后才推荐下一次补加量。([aist](https://www.aist.org/AIST/aist/AIST/Publications/Digital%20Transformations/20-jan-digital-transformation.pdf))

---

## 5.3 成分合格判断

不要只判断“是否达到目标值”，而要判断：

| 判断项 | 逻辑 |
|---|---|
| 是否低于下限 | 低于则需要补加 |
| 是否高于上限 | 高于则报警，不能靠加合金解决 |
| 是否靠近上限 | 提示风险 |
| 是否靠近下限 | 根据是否还有后续工序决定 |
| 是否预计漂移 | 考虑后续过程变化 |

输出应包括：

| 输出 | 示例 |
|---|---|
| 合格状态 | 合格 / 需补 Mn / C 风险 / Si 接近上限 |
| 推荐方案 | 硅锰 1650 kg，高碳铬铁 150 kg |
| 预计成分 | C 0.079，Si 0.226，Mn 1.185，Cr 0.395 |
| 成本 | 185.4 元/t |
| 风险说明 | C 距上限 0.001，建议人工确认 |

# 7. 当前两张图对我们的具体启发

## 启发一：不能只做单次静态计算

我们的工具要改成“分阶段计算”：

| 阶段 | 求解目标 |
|---|---|
| BOF 出钢 | 接近目标下限，低成本粗配 |
| LF 初期 | 根据样品修正，补主要元素 |
| LF 后期 | 小批量精调，避免超标 |
| 完成后 | 回写数据库，校准回收率 |

---

## 启发二：回收率必须从“固定参数”升级为“动态参数”

推荐回收率计算逻辑：

$$
\eta_{final}=w_1\eta_{default}+w_2\eta_{same\ grade}+w_3\eta_{similar\ heats}+w_4\eta_{ML}
$$

第一版可以简化为：

$$
\eta_{final}=0.5\eta_{default}+0.3\eta_{same\ grade}+0.2\eta_{recent}
$$

等历史数据足够，再加入 ML 模型。

---

## 启发三：系统必须有历史成本对比

第二张图里有“当前钢种历史炉次成本统计：最小值、平均值、最大值”。这个非常实用。

我们的系统应该输出：

| 指标 | 作用 |
|---|---|
| 本炉理论最低成本 | 看优化结果 |
| 本炉执行成本 | 现场实际成本 |
| 同钢种近 30 炉平均成本 | 判断是否改善 |
| 同钢种历史最低成本 | 判断节约潜力 |
| 系统推荐 vs 人工方案差额 | 量化收益 |

AIST/SSAB 的案例显示，模型上线后可实现合金成本节约，并减少部分化验次数；其报告中列举了不同钢种/场景下的单炉节约案例。([aist](https://www.aist.org/AIST/aist/AIST/Publications/Digital%20Transformations/20-jan-digital-transformation.pdf))

# 8. 最终推荐方案

结合 PPT、流程图和界面图，我建议我们的方案定位为：

> **热卷合金自动计算与成本寻优系统：以线性/整数规划为核心，以动态回收率模型为校正，以 BOF/LF 多阶段闭环为流程，以数据库为支撑。**

技术路线：

| 阶段 | 内容 | 算法 |
|---|---|---|
| V1 | 手工录入 + 固定回收率 + LP/MIP 计算 | 线性规划/整数规划 |
| V2 | 接入炉次、化验、价格、合金数据库 | LP/MIP + 规则引擎 |
| V3 | 历史炉次回收率反算 | 加权平均/相似炉次 |
| V4 | 动态回收率预测 | 聚类 + 回归/树模型 |
| V5 | 在线闭环优化 | 多阶段滚动优化 |

上线顺序不要反过来。不要一开始就搞 BP 神经网络，否则容易出现“模型看起来高级，但现场不敢用”的问题。

## 结论

这两张图不仅有参考意义，而且应该成为我们下一版工具的主要蓝本。

最重要的优化点是：

1. **从单次计算改成 BOF/LF 多阶段闭环计算；**
2. **从固定回收率改成动态回收率；**
3. **从 PPT 顺序 IF 公式改成 LP/MIP 全局成本寻优；**
4. **从只输出单耗改成输出“方案、成本、成分预测、风险解释、历史对比”；**
5. **从静态 Excel 表改成数据库驱动的小系统。**

真正贴合现场的公式不是某一个复杂 IF，而是：

> **质量平衡公式 + 成分上下限约束 + 碳/有害元素硬约束 + 回收率稳健修正 + 合金批量整数约束 + 成本最小目标函数。**

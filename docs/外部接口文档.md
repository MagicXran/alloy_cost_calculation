# 合金成本测算系统外部接口文档

版本：v1.6  
状态：正式版  
适用范围：合金成本测算系统与第三方系统之间的基础数据读取和结果推送

## 1. 文档说明

合金成本测算系统每天凌晨 `01:00` 主动同步最新基础数据，完成炼钢牌号级计算，并将结果批量推送至第三方系统。

本文档定义三类接口：

1. 牌号基础数据接口
2. 合金价格接口
3. 计算结果推送接口

## 2. 通用约定

### 2.1 请求与响应格式

```http
Content-Type: application/json; charset=utf-8
Accept: application/json
```

### 2.2 时间格式

统一采用 ISO 8601 格式：

```text
2026-07-01T01:03:20+08:00
```

### 2.3 单位约定

| 字段类型 | 单位 | 示例 | 说明 |
|---|---:|---:|---|
| 目标成分 | % | `0.16` | 表示 `0.16%` |
| 终点成分 | % | `0.07` | 表示 `0.07%` |
| 合金价格 | 元/t | `5088` | 每吨合金价格 |
| 合金投用量 | kg/t | `4.35` | 每吨钢投加量 |
| 炉重 | t | `150` | 可选字段 |
| 铝消耗 | kg/t | `1.25` | 仅审计，不参与计算 |

### 2.4 统一失败响应

读取接口失败时返回 `success=false` 和错误说明。

```json
{
  "success": false,
  "message": "最新数据尚未生成",
  "timestamp": "2026-07-01T01:00:10+08:00"
}
```

## 3. 接口一：牌号基础数据接口

### 3.1 接口说明

提供最新炼钢牌号基础数据，包括炼钢牌号、目标成分、C/Mn 终点成分、Al 铝消耗和可选炉重。

其中：

- 目标成分用于生成计算约束。
- `endpoint.C` 与 `endpoint.Mn` 参与计算。
- `alConsumptionKgPerTon` 仅用于审计，不进入计算结果。

### 3.2 请求

```http
GET /external/api/alloy-cost/grade-basis
```

### 3.3 响应示例

```json
{
  "success": true,
  "generatedAt": "2026-07-01T01:00:00+08:00",
  "records": [
    {
      "steelGrade": "Q355B",
      "target": {
        "C": 0.16,
        "Si": 0.10,
        "Mn": 0.90,
        "P": 0.018,
        "S": 0.010
      },
      "endpoint": {
        "C": 0.07,
        "Mn": 0.11
      },
      "alConsumptionKgPerTon": 1.25,
      "heatWeightT": 150,
      "remark": "牌号级默认数据"
    }
  ]
}
```

### 3.4 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `success` | boolean | 是 | 接口是否成功 |
| `generatedAt` | string | 是 | 数据生成时间 |
| `records` | array | 是 | 牌号数据列表 |
| `records[].steelGrade` | string | 是 | 炼钢牌号，必须唯一 |
| `records[].target` | object | 是 | 目标成分对象，缺失字段表示不约束 |
| `records[].target.C` | number | 否 | C 目标值，单位 `%` |
| `records[].target.Si` | number | 否 | Si 目标值，单位 `%` |
| `records[].target.Mn` | number | 否 | Mn 目标值，单位 `%` |
| `records[].target.P` | number | 否 | P 目标值，单位 `%` |
| `records[].target.S` | number | 否 | S 目标值，单位 `%` |
| `records[].endpoint` | object | 是 | 终点成分对象 |
| `records[].endpoint.C` | number | 是 | C 终点成分，单位 `%` |
| `records[].endpoint.Mn` | number | 是 | Mn 终点成分，单位 `%` |
| `records[].alConsumptionKgPerTon` | number | 否 | Al 铝消耗，单位 `kg/t`，仅审计 |
| `records[].heatWeightT` | number | 否 | 炉重，单位 `t`，仅用于总量换算 |
| `records[].remark` | string | 否 | 备注 |

## 4. 接口二：合金价格接口

### 4.1 接口说明

提供最新合金价格信息。系统默认使用 `referencePrice` 参与计算，`marketPrice` 和 `inventoryPrice` 仅作扩展保留。

合金基础信息、合金成分、启用状态和投加上限由合金成本测算系统内部手动维护，不通过外部接口同步。

### 4.2 请求

```http
GET /external/api/alloy-cost/alloy-prices
```

### 4.3 响应示例

```json
{
  "success": true,
  "generatedAt": "2026-07-01T01:00:00+08:00",
  "records": [
    {
      "alloyCode": "SIMN",
      "alloyName": "硅锰",
      "referencePrice": 5088,
      "marketPrice": 5120,
      "inventoryPrice": 5000,
      "remark": "引用价格用于计算"
    }
  ]
}
```

### 4.4 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `alloyCode` | string | 是 | 合金编码，作为唯一匹配主键 |
| `alloyName` | string | 否 | 合金名称，用于展示和人工核对 |
| `referencePrice` | number | 是 | 引用价格，单位 `元/t`，默认用于计算 |
| `marketPrice` | number | 否 | 市场价，单位 `元/t` |
| `inventoryPrice` | number | 否 | 库存价，单位 `元/t` |
| `remark` | string | 否 | 备注 |

## 5. 接口三：计算结果推送接口

### 5.1 接口说明

合金成本测算系统在完成批量计算后，主动将本批结果推送给第三方系统。

推送内容包含：

- 批次信息
- 批次汇总
- 每个炼钢牌号的计算结果
- 合金投用量
- 成分校核

`Al` 不出现在结果推送的计算口径中，也不作为最终合金消耗和合金成本的一部分。

### 5.2 请求

```http
POST /external/api/alloy-cost/results
```

### 5.3 请求示例

```json
{
  "batchId": "ALLOY-CALC-20260701-0100-001",
  "calculatedAt": "2026-07-01T01:03:20+08:00",
  "summary": {
    "total": 2,
    "success": 1,
    "failed": 1
  },
  "results": [
    {
      "steelGrade": "Q355B",
      "status": "OK",
      "costPerTon": 36.82,
      "totalKgPerTon": 7.21,
      "heatWeightT": 150,
      "totalHeatKg": 1081.5,
      "totalHeatCost": 5523.0,
      "alloys": [
        {
          "alloyCode": "SIMN",
          "alloyName": "硅锰",
          "alloyType": "锰系合金",
          "kgPerTon": 4.35,
          "costPerTon": 22.13,
          "heatKg": 652.5,
          "referencePrice": 5088
        }
      ],
      "chemistry": {
        "C": 0.158,
        "Si": 0.101,
        "Mn": 0.902
      },
      "chemistryChecks": [
        {
          "element": "Mn",
          "target": 0.90,
          "predictedValue": 0.902
        }
      ],
      "warnings": []
    },
    {
      "steelGrade": "Q460C",
      "status": "FAILED",
      "errors": [
        {
          "message": "当前目标成分、终点成分和合金库约束下 LP 不可行。"
        }
      ]
    }
  ]
}
```

### 5.4 顶层字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `batchId` | string | 是 | 批次号，也是幂等键 |
| `calculatedAt` | string | 是 | 计算完成时间 |
| `summary` | object | 是 | 批次汇总 |
| `summary.total` | number | 是 | 总牌号数 |
| `summary.success` | number | 是 | 成功数 |
| `summary.failed` | number | 是 | 失败数 |
| `results` | array | 是 | 每个炼钢牌号的结果 |

### 5.5 `results[]` 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `steelGrade` | string | 是 | 炼钢牌号 |
| `status` | string | 是 | `OK` 或 `FAILED` |
| `costPerTon` | number | 成功时必填 | 吨钢成本，单位 `元/t`，不含 Al |
| `totalKgPerTon` | number | 成功时必填 | 合金总投用量，单位 `kg/t`，不含 Al |
| `heatWeightT` | number | 否 | 炉重，单位 `t` |
| `totalHeatKg` | number | 有炉重时填 | 本炉总投用量，单位 `kg` |
| `totalHeatCost` | number | 有炉重时填 | 本炉总成本，单位 `元` |
| `alloys` | array | 成功时必填 | 合金投用明细，不含 Al |
| `chemistry` | object | 成功时必填 | 预测最终成分 |
| `chemistryChecks` | array | 成功时必填 | 目标值与预测值对照 |
| `warnings` | array | 否 | 非阻断告警 |
| `errors` | array | 失败时必填 | 失败原因 |

### 5.6 `results[].alloys[]` 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `alloyCode` | string | 是 | 合金编码 |
| `alloyName` | string | 是 | 合金名称 |
| `alloyType` | string | 否 | 合金种类 |
| `kgPerTon` | number | 是 | 投用量，单位 `kg/t` |
| `costPerTon` | number | 是 | 成本贡献，单位 `元/t` |
| `heatKg` | number | 有炉重时填 | 本炉投用量，单位 `kg` |
| `referencePrice` | number | 是 | 引用价格，单位 `元/t` |

### 5.7 `results[].chemistryChecks[]` 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `element` | string | 是 | 元素名 |
| `target` | number | 是 | 目标值 |
| `predictedValue` | number | 是 | 预测值 |

### 5.8 `results[].errors[]` 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `message` | string | 是 | 错误说明 |

### 5.9 推送规则

- `batchId` 必须幂等。
- 成功项和失败项必须一起推送。
- 成功项必须返回 `alloys`、`chemistry` 和 `chemistryChecks`。
- 失败项必须返回 `errors`。
- `chemistryChecks` 只保留 `element`、`target`、`predictedValue`。

## 6. 第三方 ACK 响应

```json
{
  "success": true,
  "batchId": "ALLOY-CALC-20260701-0100-001",
  "receivedCount": 2,
  "receivedAt": "2026-07-01T01:03:25+08:00"
}
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `success` | boolean | 是 | 是否成功 |
| `batchId` | string | 是 | 对应批次号 |
| `receivedCount` | number | 是 | 接收条数 |
| `receivedAt` | string | 是 | 接收时间 |

## 7. 调用时序

1. 合金成本测算系统在每天凌晨 `01:00` 读取牌号基础数据。
2. 合金成本测算系统读取合金价格信息。
3. 合金成本测算系统使用内部手动维护的合金基础信息执行计算。
4. 合金成本测算系统将结果批量推送给第三方系统。
5. 第三方系统返回 ACK。

## 8. 对接说明

- 牌号基础数据接口的目标成分按单值表达，缺失字段表示不约束。
- 合金基础信息由合金成本测算系统内部手动维护，暂不提供外部同步接口。
- 合金价格接口使用 `alloyCode` 匹配系统内部合金基础信息，`referencePrice` 为计算口径。
- 计算结果推送接口中的 `Al` 不参与计算、不计消耗、不计成本。
- 若某个炼钢牌号计算失败，失败结果仍需推送，便于第三方系统进行补偿和对账。

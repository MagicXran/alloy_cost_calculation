# 结论先说

你的方向是对的：**LP 线性规划是合金成本最优组合的正确数学起点**。

但如果你想做成一个真正可落地、可推广、可解释的“合金成本优化系统”，而不是 Excel 计算器，我建议目标模型不要叫简单 LP，而应定义为：

> **多元素、多合金、带工艺安全裕量的最小成本配料优化模型**  
> 第一阶段用 **LP** 求理论连续最优；  
> 第二阶段升级为 **MILP/整数规划** 处理包装、最小投料量、是否启用某种合金、现场规则；  
> 第三阶段加入 **鲁棒优化/安全裕量**，防止回收率、终点成分、合金成分波动导致成分越界。

更直白地说：

```text
只算理论最低成本：LP 足够。
要生成现场可执行投料单：MILP 更合适。
要保证现场稳定命中：MILP + 鲁棒约束/安全裕量。
要长期越用越准：再叠加历史数据学习回收率和命中率。
```

LP/MILP 不是落后方法。对于这类“原料配比 + 成分约束 + 成本最低”的问题，只要成本、成分贡献、回收率都是线性的，**数学优化求解器比遗传算法、神经网络、启发式规则更适合作为核心求解器**。线性优化的定义就是在一组线性关系约束下计算最优解，SciPy 的 `linprog` 也是“最小化线性目标函数，并满足线性等式/不等式约束”的标准形式。([Google for Developers](https://developers.google.com/optimization/lp))

# 1. “最先进最优”的合金成本模型应该长什么样？

不是一个单一算法，而是一个分层模型：

```text
第 1 层：确定性 LP 模型
    已知目标成分、终点成分、回收率、合金价格、合金成分
    求理论最低成本 kg/t 方案

第 2 层：MILP 模型
    加入包装规格、最小投料量、整数包数、启用/禁用材料、替代料逻辑
    求现场可执行最低成本方案

第 3 层：鲁棒优化模型
    考虑回收率误差、终点成分误差、合金成分波动
    求更安全的最低成本方案

第 4 层：闭环学习模型
    用历史炉次修正回收率、安全裕量、不同钢种/不同工艺路径的实际命中率

第 5 层：多目标决策模型
    成本最低 + 成分命中 + 少用料种 + 操作简单 + 风险可控
```

在你当前限定条件下：

> 不考虑外部系统；终点元素、回收率手动维护；合金价格和合金成分可获取。

最合理的先进版本是：

```text
确定性 LP/MILP + 人工安全裕量 + 方案解释 + 敏感性分析
```

暂时不应该一上来做神经网络、强化学习、贝叶斯优化。那些东西更适合在你有大量真实炉次数据之后，用来预测回收率、命中率、风险，而不是替代数学优化器。

已有研究也有用聚类、决策树、线性回归、梯度提升等方法做铁合金消耗建模和推荐，但它们依赖历史传感器/炉次数据，核心价值是“预测实际反应和消耗”，不是在已知线性约束下替代 LP/MILP。([arXiv](https://arxiv.org/abs/2204.07421))

# 2. 纯 LP 是否足够？

## 2.1 纯 LP 适用的前提

如果满足下面条件，LP 就是最优解：

| 条件 | 是否符合你当前设想 |
|---|---|
| 合金加入量可以是连续变量 | 理论测算时可以 |
| 合金价格固定 | 可以 |
| 合金成分固定 | 可以 |
| 回收率固定 | 可以 |
| 终点成分固定 | 可以 |
| 成分约束是上下限 | 可以 |
| 不考虑包装取整 | 理论测算可以 |
| 不考虑最小投料量 | 理论测算可以 |
| 不考虑库存和料仓 | 当前可以 |
| 不考虑复杂反应非线性 | 当前可以 |

那么模型就是标准 LP：

```text
目标：合金成本最低

约束：
最终 C 在范围内
最终 Si 在范围内
最终 Mn 在范围内
最终 Cr 在范围内
最终 V 在范围内
最终 Nb 在范围内
...
每种合金加入量 ≥ 0
```

这种情况下，LP 能给出全局最优解，不需要遗传算法、粒子群、模拟退火之类的方法。那些方法对非凸、黑箱、不可微问题有价值，但对线性配料问题通常是降级，因为它们一般只能给近似解，且难以证明全局最优。

---

## 2.2 但纯 LP 的现场问题很明显

纯 LP 算出来的结果可能是：

```text
硼铁：0.037 kg/t
钛铁：0.142 kg/t
高碳锰铁：1.873 kg/t
低碳锰铁：0.294 kg/t
硅锰：3.681 kg/t
```

理论上最优，但现场可能不方便。

主要问题有 8 类：

| 问题 | 说明 |
|---|---|
| 投料量不是整数 | 现场按 25 kg、50 kg、吨袋、料仓下料，不可能无限精细 |
| 出现很小投料量 | 比如某合金只加 2 kg/炉，现场可能不执行 |
| 过度贴边 | 为省钱，模型会让昂贵元素贴近下限，风险变高 |
| 上限风险 | 便宜高碳合金可能把 C、Si、P、S、N 等顶到上限边缘 |
| 启用材料过多 | 最低成本方案可能用了 8 种料，操作复杂 |
| 无法表达工艺禁忌 | 比如某钢种禁用某合金，某阶段不能加某料 |
| 无法表达替代关系 | 高碳锰铁、中碳锰铁、低碳锰铁之间有逻辑选择 |
| 无法处理安全裕量 | 回收率和终点成分手动维护，本身就有误差 |

所以：**LP 是理论最优，不等于现场最优。**

# 3. 推荐的核心模型：先 LP，后 MILP

## 3.1 决策变量

设：

```text
x_j = 第 j 种合金加入量，单位 kg/t钢
```

例如：

| 变量 | 含义 |
|---|---|
| x_硅锰 | 硅锰 kg/t |
| x_高碳锰铁 | 高碳锰铁 kg/t |
| x_中碳锰铁 | 中碳锰铁 kg/t |
| x_低碳锰铁 | 低碳锰铁 kg/t |
| x_硅铁 | 硅铁 kg/t |
| x_钒铁 | 钒铁 kg/t |
| x_铌铁 | 铌铁 kg/t |
| x_钛铁 | 钛铁 kg/t |
| x_高碳铬铁 | 高碳铬铁 kg/t |
| x_低碳铬铁 | 低碳铬铁 kg/t |
| x_镍板 | 镍板 kg/t |
| x_铜板 | 铜板 kg/t |
| x_钼铁 | 钼铁 kg/t |
| x_硼铁 | 硼铁 kg/t |

---

## 3.2 输入参数

每个合金要有完整成分表，不要只维护“主元素”。

例如硅锰不能只维护 Si、Mn，还要维护 C、P、S，甚至 N。

| 参数 | 含义 |
|---|---|
| `p_j` | 第 j 种合金价格，元/t合金 |
| `a_ij` | 第 j 种合金中第 i 个元素含量 |
| `r_ij` | 第 j 种合金中第 i 个元素的回收率 |
| `b_i` | 钢水当前已有第 i 个元素含量，也就是终点/残余成分 |
| `L_i` | 第 i 个元素目标下限 |
| `U_i` | 第 i 个元素目标上限 |
| `x_j_min` | 第 j 种合金最小可加量 |
| `x_j_max` | 第 j 种合金最大可加量 |
| `W` | 炉次钢水量，t |
| `pack_j` | 第 j 种合金包装/投料粒度，kg |

# 4. 成分贡献公式

如果：

```text
x_j：kg/t钢
a_ij：合金中元素质量分数，例如 65% 写成 0.65
r_ij：回收率，例如 95% 写成 0.95
b_i：当前钢水中元素含量，单位 %
```

那么第 i 个元素最终预测值：

```text
final_i = b_i + Σ(x_j × a_ij × r_ij / 10)
```

为什么除以 10？

因为：

```text
1 kg/t = 1 kg / 1000 kg钢 = 0.1%
```

举例：

```text
加 1 kg/t 硅锰
硅锰 Mn = 65% = 0.65
Mn 回收率 = 95% = 0.95
```

进入钢水的 Mn：

```text
1 × 0.65 × 0.95 = 0.6175 kg/t
```

折算成百分含量：

```text
0.6175 / 1000 × 100% = 0.06175%
```

也就是：

```text
1 × 0.65 × 0.95 / 10 = 0.06175%
```

如果你的数据库里 `a_ij` 是 `65` 而不是 `0.65`，公式要改成：

```text
final_i = b_i + Σ(x_j × a_ij × r_ij / 1000)
```

前提是 `r_ij` 用 `0.95` 表示。

# 5. 最基础 LP 模型

## 5.1 目标函数

```text
minimize Cost = Σ(p_j × x_j / 1000)
```

其中：

```text
p_j：元/t合金
x_j：kg/t钢
```

所以：

```text
p_j × x_j / 1000 = 元/t钢
```

例如：

```text
硅锰价格 = 6000 元/t
硅锰单耗 = 5 kg/t钢

成本 = 6000 × 5 / 1000 = 30 元/t钢
```

---

## 5.2 成分上下限约束

对每个元素 i：

```text
L_i ≤ b_i + Σ(x_j × a_ij × r_ij / 10) ≤ U_i
```

例如 Mn：

```text
Mn下限 ≤ 终点Mn + 各合金带入Mn ≤ Mn上限
```

C：

```text
C下限 ≤ 终点C + 各合金带入C ≤ C上限
```

Si：

```text
Si下限 ≤ 终点Si + 各合金带入Si ≤ Si上限
```

---

## 5.3 非负约束

```text
x_j ≥ 0
```

必要时加最大加入量：

```text
x_j ≤ x_j_max
```

# 6. 但更推荐你直接设计成 MILP

MILP，也就是混合整数线性规划，适合处理“连续投料量 + 整数包数 + 是否使用某种材料”的问题。OR-Tools 对 MIP 的说明也是：在线性约束基础上，要求部分变量必须为整数；其文档给出的求解流程包括定义变量、定义约束、定义目标、调用 MIP 求解器。([Google for Developers](https://developers.google.com/optimization/mip/mip_example)) CBC 这类开源求解器也专门用于混合整数规划，采用 branch-and-cut 一类算法处理整数变量。([COIN-OR](https://coin-or.github.io/Cbc/intro.html))

## 6.1 是否启用某种合金

引入二进制变量：

```text
y_j = 1 表示使用第 j 种合金
y_j = 0 表示不使用第 j 种合金
```

约束：

```text
x_j ≤ M_j × y_j
```

含义：

```text
如果 y_j = 0，则 x_j 必须为 0
如果 y_j = 1，则 x_j 可以在允许范围内取值
```

如果现场规定某合金一旦使用就不能太少：

```text
x_j ≥ min_j × y_j
```

例如：

```text
只要用了硼铁，至少加 0.05 kg/t
```

---

## 6.2 包装规格 / 投料粒度

假设一炉钢水量：

```text
W = 150 t
```

某合金一袋：

```text
pack_j = 25 kg
```

引入整数变量：

```text
n_j = 第 j 种合金投料袋数
```

则：

```text
x_j × W = pack_j × n_j
```

等价于：

```text
x_j = pack_j × n_j / W
```

这时求出来的就不是理论连续 kg/t，而是现场可执行投料量。

例如：

```text
n_硅锰 = 24 袋
pack_硅锰 = 25 kg
W = 150 t

x_硅锰 = 24 × 25 / 150 = 4 kg/t
```

这比 LP 算出 `3.87 kg/t` 更接近现场实际。

---

## 6.3 限制使用料种数量

现场不一定喜欢一个方案用了太多合金。可以限制：

```text
Σ y_j ≤ K
```

比如：

```text
最多启用 6 种合金
```

或者不硬限制，而是在目标函数里惩罚：

```text
minimize 合金成本 + λ × 使用合金种类数
```

即：

```text
minimize Σ(p_j × x_j / 1000) + λ × Σy_j
```

这样模型会倾向于少用料种。

---

## 6.4 高碳/低碳合金替代逻辑

例如锰的来源有：

```text
硅锰
高碳锰铁
中碳锰铁
低碳锰铁
金属锰
```

纯 LP 会自动根据成本和元素约束选择组合。

但如果你想体现工艺规则，可以加逻辑约束。

例如某些低碳钢不允许使用高碳锰铁：

```text
y_高碳锰铁 = 0
```

或者：

```text
目标C ≤ 0.08% 时，禁用高碳锰铁
```

或者：

```text
高碳锰铁和金属锰不能同时启用
```

可写成：

```text
y_高碳锰铁 + y_金属锰 ≤ 1
```

这类逻辑是 LP 做不好的，需要 MILP。

# 7. 关键不是“目标值”，而是“上下限 + 安全裕量”

这是你们做系统时最容易踩坑的地方。

## 7.1 不要把目标成分当成等式

错误写法：

```text
最终 Mn = 1.20%
最终 Si = 0.25%
最终 C = 0.16%
```

这种写法很容易导致：

```text
模型无解
或者为了精确命中某个点，算出非常奇怪的合金组合
```

正确做法是：

```text
Mn_min ≤ 最终Mn ≤ Mn_max
Si_min ≤ 最终Si ≤ Si_max
C_min ≤ 最终C ≤ C_max
```

再设置一个“推荐目标点”或“内控目标点”，用于解释和偏差惩罚。

---

## 7.2 成本最低模型天然会贴下限

如果 V、Nb、Mo、Ni 很贵，模型一定会倾向于：

```text
V 刚好 ≥ V下限
Nb 刚好 ≥ Nb下限
Mo 刚好 ≥ Mo下限
Ni 刚好 ≥ Ni下限
```

这在理论上对，但现场有风险。

因为：

```text
回收率稍低
终点残余偏低
合金成分低于标称
称量误差
```

都会导致最终成分低于下限。

所以先进模型必须引入安全裕量。

例如：

```text
V_min = 0.030%
安全裕量 = 0.002%

优化约束不是：
V ≥ 0.030%

而是：
V ≥ 0.032%
```

对上限元素反过来：

```text
C_max = 0.180%
安全裕量 = 0.005%

优化约束不是：
C ≤ 0.180%

而是：
C ≤ 0.175%
```

# 8. 比普通 LP 更好的方案：鲁棒 LP/MILP

在你们当前“不接外部系统、参数手工维护”的条件下，最值得做的高级能力不是 AI，而是：

> **鲁棒优化。**

因为回收率、终点元素、合金成分都不是绝对准确的。

## 8.1 把参数从“单点值”改成“区间值”

例如：

```text
Mn回收率 = 95%
```

不要只维护一个值，而是维护：

```text
Mn回收率名义值 = 95%
Mn回收率下限 = 92%
Mn回收率上限 = 97%
```

终点 C 也类似：

```text
终点C名义值 = 0.060%
终点C低值 = 0.055%
终点C高值 = 0.065%
```

合金成分也类似：

```text
硅锰 Mn 名义值 = 65%
硅锰 Mn 下限 = 63%
硅锰 Mn 上限 = 67%
```

---

## 8.2 对下限元素，用最不利情况计算

例如 Mn 要保证不低于下限，就用：

```text
终点Mn低值 + 合金Mn成分低值 × Mn回收率低值
```

约束：

```text
Mn_final_worst_low ≥ Mn_min
```

也就是：

```text
终点Mn_low + Σ(x_j × Mn含量_low × Mn回收率_low / 10) ≥ Mn_min
```

---

## 8.3 对上限元素，也用最不利情况计算

例如 C 不能超上限，就用：

```text
终点C高值 + 合金C成分高值 × C回收率高值
```

约束：

```text
C_final_worst_high ≤ C_max
```

也就是：

```text
终点C_high + Σ(x_j × C含量_high × C回收率_high / 10) ≤ C_max
```

---

## 8.4 这个鲁棒模型仍然是线性的

因为你只是把系数从名义值换成保守值：

```text
普通系数：a_ij × r_ij
鲁棒系数：a_ij_low × r_ij_low 或 a_ij_high × r_ij_high
```

变量还是 `x_j`，目标函数还是线性的。

所以它仍然可以用 LP/MILP 求解，不需要复杂非线性算法。

这是非常适合你们当前阶段的“高级但可落地”的方案。

---

# 9. 最终推荐的优化目标：不要只有“成本最低”

如果只做：

```text
minimize 合金成本
```

模型会非常激进。

更合理的是分层目标。

## 9.1 第一优先级：不能违反硬约束

```text
成分不能超内控上下限
禁用材料不能用
最大加入量不能超过
包装规则不能违反
```

---

## 9.2 第二优先级：成本最低

```text
minimize Σ(p_j × x_j / 1000)
```

---

## 9.3 第三优先级：尽量靠近目标点

加入偏差惩罚：

```text
minimize 成本 + λ × Σ|final_i - target_i|
```

如果用绝对值，可以线性化，仍然是 LP/MILP。

也可以用平方偏差：

```text
minimize 成本 + λ × Σ(final_i - target_i)^2
```

这就变成 QP，二次规划。

但我建议第一版不要上 QP，先用绝对偏差，保持线性。

---

## 9.4 第四优先级：少用料种，少用小投料

```text
minimize 成本 + λ1 × 成分偏差 + λ2 × 使用料种数量 + λ3 × 小投料惩罚
```

完整目标可以写成：

```text
minimize
    Σ(p_j × x_j / 1000)
  + λ1 × Σ成分偏差
  + λ2 × Σy_j
  + λ3 × Σ小投料惩罚
```

但工程上更推荐做成“分层求解”：

```text
第一步：找所有满足成分和工艺约束的方案
第二步：在可行方案中找成本最低
第三步：在成本不超过最低成本 + 0.5 元/t 的方案中，找更稳、更简单的方案
```

这个思路比单一目标函数更贴近现场。

---

# 10. 为什么“近似最优方案”很重要？

真正现场不一定要绝对最低的那个方案。

比如系统算出两个方案：

| 方案 | 成本 | 风险 | 操作复杂度 |
|---|---:|---|---|
| A | 86.20 元/t | C、Mn 都贴边 | 用 8 种料 |
| B | 86.75 元/t | 成分更居中 | 用 5 种料 |
| C | 87.10 元/t | 命中率最高 | 用 4 种料 |

如果只看成本，选 A。

但现场可能更愿意选 B 或 C。

所以系统要输出：

```text
最低成本方案
推荐稳健方案
少料种方案
低碳风险方案
高命中率方案
```

而不是只输出一个答案。

Pyomo 这类建模工具本身就支持构建和分析优化模型，并且文档中也包含近似最优方案、不可行诊断、敏感性分析等优化分析方向，这类能力很适合做成你们系统的“解释层”。([Pyomo Documentation](https://pyomo.readthedocs.io/en/stable/))

# 12. 我建议的最终数学模型

## 12.1 集合

```text
E = 元素集合，例如 C, Si, Mn, P, S, Cr, Ni, Cu, Mo, V, Nb, Ti, B, Al, N
M = 合金集合，例如 硅锰、高碳锰铁、低碳锰铁、硅铁、钒铁等
```

---

## 12.2 参数

```text
p_j      = 合金 j 的价格，元/t
a_ij     = 合金 j 中元素 i 的含量，质量分数
r_ij     = 元素 i 在合金 j 中的回收率
b_i      = 钢水中元素 i 的已有含量，%
L_i      = 元素 i 的控制下限，%
U_i      = 元素 i 的控制上限，%
T_i      = 元素 i 的目标值，%
W        = 炉次钢水量，t
pack_j   = 合金 j 的包装规格，kg
```

---

## 12.3 变量

连续版本：

```text
x_j ≥ 0
```

其中：

```text
x_j = 合金 j 的加入量，kg/t钢
```

整数版本：

```text
n_j ∈ 整数
y_j ∈ {0,1}
```

其中：

```text
n_j = 合金 j 的投料包数
y_j = 是否使用合金 j
```

---

## 12.4 成分预测

```text
final_i = b_i + Σ(x_j × a_ij × r_ij / 10)
```

---

## 12.5 成分约束

```text
L_i ≤ final_i ≤ U_i
```

如果加入安全裕量：

```text
L_i + margin_i_low ≤ final_i ≤ U_i - margin_i_high
```

---

## 12.6 成本目标

```text
minimize Cost = Σ(p_j × x_j / 1000)
```

---

## 12.7 包装约束

```text
x_j × W = pack_j × n_j
```

---

## 12.8 是否启用约束

```text
x_j ≤ x_j_max × y_j
x_j ≥ x_j_min × y_j
```

---

## 12.9 使用料种数量约束

```text
Σy_j ≤ K
```

---

## 12.10 偏差惩罚

定义：

```text
d_i_pos ≥ final_i - T_i
d_i_neg ≥ T_i - final_i
d_i_pos ≥ 0
d_i_neg ≥ 0
```

目标函数改为：

```text
minimize
    Σ(p_j × x_j / 1000)
  + λ × Σw_i × (d_i_pos + d_i_neg)
```

其中：

```text
w_i = 元素重要性权重
λ = 成分偏差惩罚系数
```

这样仍然是线性模型。

# 13. 对你问题的直接回答：有没有比 LP 更好的方案？

## 答案是：有，但不是取代 LP，而是增强 LP。

| 方案 | 是否推荐 | 原因 |
|---|---|---|
| 纯 LP | 推荐作为理论成本基线 | 快、稳、可解释、全局最优 |
| MILP | 强烈推荐作为正式产品核心 | 能处理整数包数、是否使用、现场逻辑 |
| 鲁棒 LP/MILP | 强烈推荐作为高级能力 | 能处理回收率和终点误差 |
| 目标规划 | 推荐 | 能处理成本、成分居中、少料种等多目标 |
| QP 二次规划 | 可选 | 适合追求成分靠近目标点，但第一版没必要 |
| 随机规划 | 暂缓 | 需要概率分布和大量历史数据 |
| 机器学习 | 暂缓作为主算法 | 适合预测回收率，不适合替代优化器 |
| 遗传算法/粒子群 | 不推荐作为主算法 | 对线性/MILP问题没有优势，解释性和最优性较差 |
| 强化学习 | 不推荐当前阶段 | 需要动态环境和大量闭环反馈，不适合先做理论合金成本 |

所以最优路线是：

```text
不是 LP vs 更高级算法，
而是：

LP → MILP → 鲁棒 MILP → 数据驱动回收率修正 → 智能推荐
```

# 14. 实际业务里，最关键的 6 个模型坑

## 14.1 合金不能只按主元素建模

例如高碳锰铁不能只看 Mn。

必须同时看：

```text
Mn、C、Si、P、S
```

否则模型会为了补 Mn 大量使用高碳锰铁，最后 C 超标。

---

## 14.2 所有元素都要有上下限，不只是目标元素

例如某钢种不要求加 Cr，但高碳铬铁、废钢、残余元素可能带入 Cr。

所以 Cr 也要有上限。

类似：

```text
P、S、N、Cu、Ni、Cr、Mo、Sn、As
```

有些是有害残余元素，有些是性能元素，有些是标准限制元素。

如果没有上限，优化器会钻空子。

---

## 14.3 目标成分不能只维护一个点

必须维护三套值：

```text
标准范围：国标/企标允许范围
内控范围：现场希望控制的范围
目标点：推荐瞄准值
```

优化优先使用：

```text
内控范围
```

目标点只用于偏差惩罚和方案解释。

---

## 14.4 成本最低会天然选择边界解

这不是模型错，而是优化器的本性。

所以必须做：

```text
安全裕量
近似最优方案
成分居中方案
鲁棒方案
```

否则系统容易被现场认为“算得太冒险”。

---

## 14.5 需要区分理论成本和可执行成本

理论 LP 方案：

```text
成本最低，但投料量可能不可执行
```

MILP 方案：

```text
成本略高，但可以按袋、按料仓、按最小投料量执行
```

系统里应该同时显示：

```text
理论最低成本
可执行最低成本
两者差额
```

这个差额本身就是管理价值。

---

## 14.6 不可行诊断比求解成功更重要

如果模型无解，系统不能只提示：

```text
无可行解
```

而要告诉用户：

```text
为什么无解？
是 C 上限太低？
是 Mn 下限太高？
是低碳锰铁价格缺失？
是禁用了唯一可补 Nb 的合金？
是终点C太高导致不能用高碳合金？
是包装取整后无法满足 B 范围？
```

这是产品化的核心能力。

# 1. 为什么必须建模？

如果只是 Excel 计算器，可以不用系统化建模，写公式就行。

但你们想做成通用产品，就必须建模。原因有 5 个。

---

## 1.1 Excel 公式是隐式逻辑，系统需要显式模型

Excel 里很多逻辑藏在公式里：

```text
先用硅锰？
高碳锰铁什么时候用？
低碳锰铁什么时候替代？
C 超了怎么办？
Mn 不够怎么办？
合金价格变了怎么办？
```

这些如果不建模，系统最后还是一堆硬编码。

产品化后必须把它抽象成：

```text
元素约束
合金变量
成本目标
投料规则
鲁棒裕量
方案解释
```

---

## 1.2 不同现场参数不同，但问题结构相同

不同钢厂可能不同的是：

```text
钢种成分范围不同
合金种类不同
合金价格不同
回收率不同
终点成分不同
包装规格不同
禁用规则不同
```

但底层问题是一样的：

```text
在满足成分约束的前提下，选择一组合金，使成本最低。
```

所以你们要抽象的是**普世问题结构**，不是某个 Excel 的公式。

---

## 1.3 不建模就无法求“最优”

Excel 公式一般是经验顺序：

```text
先算硅锰
再算锰铁
再算硅铁
再算微合金
```

这叫“规则计算”，不是“全局优化”。

建模之后，系统可以自动判断：

```text
现在价格下，是硅锰便宜，还是硅铁 + 锰铁更便宜？
终点 C 偏高时，还能不能用高碳锰铁？
高碳铬铁便宜，但会不会把 C 顶超？
低碳锰铁贵，但是否能换来更低风险？
```

---

## 1.4 不建模就无法做多方案

正式产品不能只输出一个答案。

应该能输出：

```text
理论最低成本方案
现场可执行方案
稳健推荐方案
少料种方案
低碳风险方案
禁用某合金替代方案
```

这些都依赖模型，不是简单公式能优雅实现的。

---

## 1.5 不建模就无法解释“为什么”

现场最关心的不是系统算了多少，而是：

```text
为什么这么算？
为什么不用更便宜的高碳锰铁？
为什么用了低碳锰铁？
为什么 Mn 只压到这个位置？
为什么这个方案比最低成本贵 1 元/t？
```

只有把目标、约束、变量、规则建清楚，才能解释。

# 5. 第三层：数学优化模型

这是核心。

你们的业务可以建成一个标准的：

```text
混合整数线性规划 MILP
```

如果加入鲁棒边界，就是：

```text
鲁棒 MILP
```

---

# 6. 基础数学建模

## 6.1 集合

```text
E = 元素集合
A = 合金集合
```

例如：

```text
E = {C, Si, Mn, P, S, Cr, Ni, Cu, Mo, V, Nb, Ti, B, Al, N}

A = {硅锰, 高碳锰铁, 中碳锰铁, 低碳锰铁, 硅铁, 钒铁, 铌铁, 钛铁, 高碳铬铁, 低碳铬铁, 镍板, 铜板, 钼铁, 硼铁}
```

---

## 6.2 决策变量

核心变量：

```text
x_j = 第 j 种合金加入量，kg/t钢
```

其中：

```text
j ∈ A
```

如果要处理是否使用某种合金，引入：

```text
y_j ∈ {0,1}
```

含义：

```text
y_j = 1，使用该合金
y_j = 0，不使用该合金
```

如果要处理包装取整，引入：

```text
n_j ∈ 整数
```

含义：

```text
n_j = 第 j 种合金投料包数
```

---

## 6.3 输入参数

| 参数 | 含义 |
|---|---|
| `p_j` | 合金 j 的价格，元/t合金 |
| `a_ij` | 合金 j 中元素 i 的含量 |
| `r_ij` | 元素 i 在合金 j 中的回收率 |
| `b_i` | 钢水中元素 i 的终点/残余成分 |
| `L_i` | 元素 i 的内控下限 |
| `U_i` | 元素 i 的内控上限 |
| `T_i` | 元素 i 的目标值 |
| `W` | 炉次钢水量，t |
| `pack_j` | 合金 j 的包装规格，kg/包 |

---

## 6.4 成分预测公式

如果：

```text
x_j：kg/t钢
a_ij：小数形式，例如 65% = 0.65
r_ij：小数形式，例如 95% = 0.95
b_i：wt%
```

那么最终预测成分：

```text
Final_i = b_i + Σ(x_j × a_ij × r_ij / 10)
```

为什么除以 10？

```text
1 kg/t = 1 kg / 1000 kg钢 = 0.1%
```

例如：

```text
加入 1 kg/t 硅锰
硅锰 Mn = 65% = 0.65
Mn 回收率 = 95% = 0.95

Mn贡献 = 1 × 0.65 × 0.95 / 10
       = 0.06175%
```

如果合金成分用 `65` 表示，而不是 `0.65`，则公式变成：

```text
Final_i = b_i + Σ(x_j × a_ij × r_ij / 1000)
```

---

## 6.5 成分约束

对每个元素：

```text
L_i ≤ Final_i ≤ U_i
```

展开：

```text
L_i ≤ b_i + Σ(x_j × a_ij × r_ij / 10) ≤ U_i
```

这就是合金优化的核心约束。

---

## 6.6 成本目标函数

```text
Minimize Cost = Σ(p_j × x_j / 1000)
```

其中：

```text
p_j：元/t合金
x_j：kg/t钢
```

所以：

```text
p_j × x_j / 1000 = 元/t钢
```

---

# 7. 现场可执行 MILP 建模

纯 LP 会算出连续值，例如：

```text
硅锰 3.728 kg/t
钒铁 0.137 kg/t
硼铁 0.021 kg/t
```

现场不一定能投。

所以要加整数约束。

---

## 7.1 包装取整约束

假设：

```text
W = 炉次钢水量，t
pack_j = 第 j 种合金包装规格，kg/包
n_j = 投料包数，整数
```

那么：

```text
x_j × W = pack_j × n_j
```

也就是：

```text
x_j = pack_j × n_j / W
```

例如：

```text
W = 150 t
硅锰一袋 25 kg
n_硅锰 = 24 袋

x_硅锰 = 25 × 24 / 150
       = 4 kg/t
```

这样算出的就是现场可以执行的方案。

---

## 7.2 是否启用某合金

```text
x_j ≤ M_j × y_j
```

其中：

```text
M_j = 该合金最大允许加入量
```

如果 `y_j = 0`，则：

```text
x_j = 0
```

如果 `y_j = 1`，则：

```text
x_j 可以大于 0
```

---

## 7.3 最小投料量约束

如果某合金一旦使用，至少要加一定量：

```text
x_j ≥ min_j × y_j
```

例如：

```text
用了钒铁，至少 0.05 kg/t
```

---

## 7.4 最多使用料种数

```text
Σ y_j ≤ K
```

例如：

```text
一炉最多使用 6 种合金
```

这个约束对现场很实用。

否则优化器可能为了省 0.1 元/t，搞出 9 种合金的小投料组合。

---

## 7.5 禁用规则

如果某钢种禁用高碳锰铁：

```text
y_高碳锰铁 = 0
```

如果某钢种禁用含硼合金：

```text
y_硼铁 = 0
```

---

## 7.6 互斥规则

例如不希望高碳锰铁和金属锰同时使用：

```text
y_高碳锰铁 + y_金属锰 ≤ 1
```

---

# 8. 鲁棒建模

这是你们正式产品最重要的提升点。

普通模型假设：

```text
终点成分准确
回收率准确
合金成分准确
```

但现场不是这样。

所以需要鲁棒模型。

---

## 8.1 对下限元素，用保守下限计算

例如 Mn、Si、V、Nb、Ti 这类元素，不能低于下限。

那就按最不利情况算：

```text
终点残余取低值
合金元素含量取低值
回收率取低值
```

约束：

```text
b_i_low + Σ(x_j × a_ij_low × r_ij_low / 10) ≥ L_i + margin_i_low
```

含义：

> 即使终点残余偏低、合金成分偏低、回收率偏低，最终成分也不能低于内控下限。

---

## 8.2 对上限元素，用保守上限计算

例如 C、P、S、N、Cu 等元素，不能超过上限。

那就按最不利情况算：

```text
终点残余取高值
合金元素含量取高值
回收率取高值
```

约束：

```text
b_i_high + Σ(x_j × a_ij_high × r_ij_high / 10) ≤ U_i - margin_i_high
```

含义：

> 即使终点 C 偏高、高碳合金 C 含量偏高、C 回收率偏高，最终 C 也不能超过上限。

---

## 8.3 同时输出名义预测

鲁棒模型求解时用保守约束，但展示时要同时显示名义预测：

```text
Final_i_nominal = b_i_nominal + Σ(x_j × a_ij_nominal × r_ij_nominal / 10)
```

最终界面显示：

| 元素 | 名义预测 | 最差下限 | 最差上限 | 内控范围 | 风险 |
|---|---:|---:|---:|---:|---|
| C | 0.162 | - | 0.175 | 0.150~0.180 | 低 |
| Mn | 1.205 | 1.163 | - | 1.150~1.250 | 中 |
| V | 0.036 | 0.032 | - | 0.030~0.050 | 中 |

这样现场才能理解：

```text
正常情况是多少？
最差情况是否还能守住边界？
```

---

# 9. 多目标建模

严格说，正式产品不应该只优化成本。

因为只优化成本会天然贴边。

---

## 9.1 基础目标

```text
成本最低
```

即：

```text
Minimize Σ(p_j × x_j / 1000)
```

---

## 9.2 成分偏离目标的惩罚

引入偏差变量：

```text
d_i_pos ≥ Final_i - T_i
d_i_neg ≥ T_i - Final_i
d_i_pos ≥ 0
d_i_neg ≥ 0
```

偏差成本：

```text
Σ w_i × (d_i_pos + d_i_neg)
```

其中：

```text
w_i = 元素重要性权重
```

例如：

```text
C 权重大
B 权重大
Nb/V/Ti 权重大
普通残余元素权重小
```

---

## 9.3 使用料种数惩罚

```text
Σ y_j
```

目标函数变成：

```text
Minimize
    合金成本
  + λ1 × 成分偏差
  + λ2 × 使用料种数
  + λ3 × 小投料惩罚
```

但工程上，我更建议不要一开始就把所有目标塞进一个公式。

更好的方式是分步求解。

---

# 10. 推荐的求解流程模型

正式产品建议这样跑：

```text
Step 1：理论 LP
Step 2：确定性 MILP
Step 3：鲁棒 MILP
Step 4：多方案生成
Step 5：方案解释
```

---

## 10.1 Step 1：理论 LP

目的：

```text
得到理论最低成本基线
```

特点：

```text
不考虑包装取整
不考虑整数袋数
不考虑鲁棒裕量
```

输出：

```text
理论最低成本：82.30 元/t
```

---

## 10.2 Step 2：确定性 MILP

目的：

```text
得到现场可执行最低成本
```

加入：

```text
包装取整
最小投料量
最大投料量
禁用规则
最多料种数
```

输出：

```text
可执行最低成本：83.10 元/t
```

差额：

```text
83.10 - 82.30 = 0.80 元/t
```

解释为：

```text
可执行性成本
```

---

## 10.3 Step 3：鲁棒 MILP

目的：

```text
得到推荐生产方案
```

加入：

```text
终点成分波动
合金成分波动
回收率波动
安全裕量
```

输出：

```text
鲁棒推荐成本：84.20 元/t
```

差额：

```text
84.20 - 83.10 = 1.10 元/t
```

解释为：

```text
鲁棒性成本
```

---

## 10.4 Step 4：多方案生成

以鲁棒推荐成本 `C*` 为基础，设置成本窗口：

```text
Cost ≤ C* + 0.50 元/t
```

然后生成不同策略的方案：

```text
最低成本方案
稳健方案
少料种方案
低碳风险方案
禁用某合金替代方案
```

---

# 11. 多方案怎么建模？

## 11.1 方案 A：最低成本方案

目标：

```text
Minimize Cost
```

用于成本基线。

---

## 11.2 方案 B：稳健推荐方案

目标：

```text
Minimize Cost + 风险惩罚
```

或者：

```text
在成本不超过最低成本 + Δ 的情况下，最大化安全距离
```

例如：

```text
Cost ≤ C* + 0.5
Maximize 最小安全距离
```

---

## 11.3 方案 C：少料种方案

约束：

```text
Cost ≤ C* + 0.8
```

目标：

```text
Minimize Σ y_j
```

含义：

```text
在成本略微增加的允许范围内，尽量少用合金种类。
```

---

## 11.4 方案 D：低碳风险方案

约束：

```text
Cost ≤ C* + 1.0
```

目标：

```text
Maximize C 距离上限的安全距离
```

或者：

```text
Minimize 高碳合金带入 C
```

适用于：

```text
低碳钢
终点 C 偏高炉次
窄 C 控制钢种
```

---

## 11.5 方案 E：禁用某合金替代方案

例如模拟硅锰缺货：

```text
y_硅锰 = 0
```

然后重新求解。

输出：

```text
不用硅锰时，成本增加多少？
换成哪些合金？
成分风险有什么变化？
```

---

# 12. 解释模型怎么建？

解释模型不是另一个求解器，而是对求解结果做结构化分析。

---

## 12.1 成本解释

输出：

| 指标 | 数值 |
|---|---:|
| 理论 LP 成本 | 82.30 元/t |
| 可执行 MILP 成本 | 83.10 元/t |
| 鲁棒推荐成本 | 84.20 元/t |
| 可执行性成本 | +0.80 元/t |
| 鲁棒性成本 | +1.10 元/t |

解释：

```text
推荐方案比理论最低方案高 1.90 元/t。
其中 0.80 元/t 来自包装取整和现场投料规则；
1.10 元/t 来自终点成分、回收率和合金成分波动下的安全裕量。
```

---

## 12.2 成分解释

输出：

| 元素 | 当前/终点 | 预测最终 | 内控范围 | 安全距离 |
|---|---:|---:|---:|---:|
| C | 0.060 | 0.162 | 0.150~0.180 | 0.018 |
| Mn | 0.120 | 1.205 | 1.150~1.250 | 0.045 |

## 12.3 元素来源解释

例如 Mn：

| 来源 | Mn 贡献 |
|---|---:|
| 终点 Mn | 0.120 |

## 12.4 约束触发解释

系统应该自动生成类似说明：

```text
C 上限约束接近触发，因此系统没有继续增加高碳锰铁，而是选择部分低碳锰铁。

Mn 下限约束接近触发，因此保留了硅锰和锰铁组合。

V 成本较高，因此系统只将 V 控制在内控下限上方安全裕量内，没有向目标上限靠近。

```

这就是方案解释。

# 17. 一个完整建模例子

假设元素只考虑：

```text
C, Si, Mn
```

合金只考虑：

```text
硅锰、高碳锰铁、低碳锰铁、硅铁
```

目标：

```text
C: 0.140% ~ 0.180%
Si: 0.180% ~ 0.260%
Mn: 1.150% ~ 1.250%
```

终点：

```text
C = 0.060%
Si = 0.010%
Mn = 0.120%
```

变量：

```text
x1 = 硅锰 kg/t
x2 = 高碳锰铁 kg/t
x3 = 低碳锰铁 kg/t
x4 = 硅铁 kg/t
```

目标函数：

```text
Minimize
    p1*x1/1000
  + p2*x2/1000
  + p3*x3/1000
  + p4*x4/1000
```

约束：

```text
0.140 ≤ C_final ≤ 0.180
0.180 ≤ Si_final ≤ 0.260
1.150 ≤ Mn_final ≤ 1.250
x1,x2,x3,x4 ≥ 0
```

其中：

```text
C_final = 0.060 + x1*C_硅锰*r_C硅锰/10
                + x2*C_高碳锰铁*r_C高碳锰铁/10
                + x3*C_低碳锰铁*r_C低碳锰铁/10
                + x4*C_硅铁*r_C硅铁/10

Si_final = 0.010 + x1*Si_硅锰*r_Si硅锰/10
                + x4*Si_硅铁*r_Si硅铁/10

Mn_final = 0.120 + x1*Mn_硅锰*r_Mn硅锰/10
                + x2*Mn_高碳锰铁*r_Mn高碳锰铁/10
                + x3*Mn_低碳锰铁*r_Mn低碳锰铁/10
```

如果加入包装：

```text
x1 * W = 25 * n1
x2 * W = 25 * n2
x3 * W = 25 * n3
x4 * W = 25 * n4

n1,n2,n3,n4 为整数
```

如果加入是否使用：

```text
x1 ≤ M1*y1
x2 ≤ M2*y2
x3 ≤ M3*y3
x4 ≤ M4*y4

y1,y2,y3,y4 ∈ {0,1}
```

这就是 MILP。

# 18. 最容易犯的建模错误

## 错误 1：把目标成分当等式

错误：

```text
Mn_final = 1.20%
```

正确：

```text
1.15% ≤ Mn_final ≤ 1.25%
```

目标值只用于偏差惩罚，不应作为硬等式。

---

## 错误 2：只维护合金主元素

错误：

```text
高碳锰铁只维护 Mn
```

正确：

```text
高碳锰铁要维护 Mn、C、Si、P、S 等。
```

否则 C、P、S 可能被优化器忽略。

---

## 错误 3：只做成本最低，不做安全裕量

成本最低会天然贴边。

必须做：

```text
安全裕量
鲁棒约束
风险提示
```

---

## 错误 4：没有单位体系

必须统一：

```text
合金加入量：kg/t钢
合金价格：元/t合金
元素含量：wt%
回收率：0~1 小数
炉重：t
包装：kg/包
```

否则很容易差 10 倍或 100 倍。

---

## 错误 5：没有无解诊断

模型无解时，系统不能只说“无解”。

要说明：

```text
哪个元素冲突？
哪个合金缺失？
哪个规则太严？
是包装取整导致无解，还是成分约束本身无解？
```

# 第一阶段：确定性 LP/MILP 优化系统

## 1.1 第一阶段的目标

第一阶段的目标不是“智能”，而是：

> **把 Excel 里的经验公式，升级为一个标准的数学优化模型。**

具体目标：

```text
1. 支持钢种成分上下限维护
2. 支持合金价格维护
3. 支持合金成分维护
4. 支持回收率维护
5. 支持终点/残余成分维护
6. 支持理论最低成本计算
7. 支持包装取整后的现场可执行投料方案
8. 输出预测成分和合金成本明细
```

第一阶段完成后，系统应该能回答：

```text
在当前钢种目标、终点成分、回收率、合金价格、合金成分已知的前提下，
到底哪组合金成本最低？
如果按现场包装规格取整，应该每炉加多少？
```

---

## 1.2 第一阶段解决什么问题？

你现在的 Excel 本质上是：

```text
固定顺序 + 固定公式 + 人工经验判断
```

比如：

```text
先算硅锰
再算高碳锰铁
再算低碳锰铁
再算硅铁
最后算 V/Nb/Ti 等微合金
```

这种方式有几个问题：

| 问题 | 说明 |
|---|---|
| 不是全局最优 | 只是按经验顺序算，不一定成本最低 |
| 对价格变化不敏感 | 合金价格变化后，最优组合可能变了 |
| 难处理副元素 | 高碳合金带入 C、P、S，Excel 可能处理不完整 |
| 难处理包装取整 | 理论 kg/t 和现场袋数之间有差异 |
| 难推广 | 每个钢厂 Excel 公式不同，系统难通用 |

第一阶段要把它变成：

```text
变量：各种合金加多少
目标：总成本最低
约束：所有元素必须满足上下限
规则：包装、最小投料量、禁用合金等
```

---

## 1.3 第一阶段的核心模型

第一阶段建议同时做两个模型：

```text
LP 理论最低成本模型
MILP 现场可执行模型
```

两者的区别：

| 模型 | 作用 |
|---|---|
| LP | 不考虑包装取整，算理论最低成本 |
| MILP | 考虑整数袋数、是否启用、最小投料量，算现场可执行方案 |

## 1.5 第一阶段的数学建模

### 1.5.1 集合

```text
E = 元素集合
A = 合金集合
```

例如：

```text
E = {C, Si, Mn, P, S, Cr, Ni, Cu, Mo, V, Nb, Ti, B, Al, N}

A = {硅锰, 高碳锰铁, 中碳锰铁, 低碳锰铁, 硅铁, 钒铁, 铌铁, 钛铁, 高碳铬铁, 低碳铬铁, 镍板, 铜板, 钼铁, 硼铁}
```

---

### 1.5.2 决策变量

连续投料变量：

```text
x_j = 第 j 种合金加入量，kg/t钢
```

是否使用合金：

```text
y_j ∈ {0,1}
```

整数包数：

```text
n_j ∈ 整数
```

其中：

```text
j ∈ A
```

---

### 1.5.3 成分预测公式

如果：

```text
x_j：kg/t钢
a_ij：合金 j 中元素 i 的质量分数，例如 65% = 0.65
r_ij：元素 i 在合金 j 中的回收率，例如 95% = 0.95
b_i：终点/残余成分，单位 wt%
```

则最终预测成分：

```text
Final_i = b_i + Σ(x_j × a_ij × r_ij / 10)
```

为什么除以 10？

```text
1 kg/t = 1 kg / 1000 kg钢 = 0.1%
```

例如：

```text
硅锰加入 1 kg/t
硅锰 Mn = 65% = 0.65
Mn 回收率 = 95% = 0.95

Mn 贡献 = 1 × 0.65 × 0.95 / 10
        = 0.06175%
```

---

### 1.5.4 成分约束

对每个元素：

```text
L_i ≤ Final_i ≤ U_i
```

展开：

```text
L_i ≤ b_i + Σ(x_j × a_ij × r_ij / 10) ≤ U_i
```

其中：

```text
L_i = 内控下限
U_i = 内控上限
```

---

### 1.5.5 成本目标函数

```text
Minimize Cost = Σ(p_j × x_j / 1000)
```

其中：

```text
p_j = 合金 j 的价格，元/t合金
x_j = 合金 j 的加入量，kg/t钢
```

所以：

```text
p_j × x_j / 1000 = 元/t钢
```

---

### 1.5.6 包装取整约束

如果：

```text
W = 炉次钢水量，t
pack_j = 第 j 种合金包装规格，kg/包
n_j = 包数
```

则：

```text
x_j × W = pack_j × n_j
```

例如：

```text
W = 150 t
硅锰一袋 25 kg
n_硅锰 = 24 袋

x_硅锰 = 25 × 24 / 150
       = 4 kg/t
```

这样模型输出的是现场可执行袋数。

---

### 1.5.7 是否启用某合金

```text
x_j ≤ M_j × y_j
```

如果：

```text
y_j = 0
```

则：

```text
x_j = 0
```

如果：

```text
y_j = 1
```

则：

```text
x_j 可以大于 0
```

---

### 1.5.8 最小投料量

如果某合金一旦使用就至少投一定量：

```text
x_j ≥ min_j × y_j
```

例如：

```text
用了钒铁，至少加 0.05 kg/t
```

## 1.8 第一阶段最容易踩的坑

### 坑 1：只维护合金主元素

例如只维护：

```text
硅锰：Si、Mn
高碳锰铁：Mn
高碳铬铁：Cr
```

这是危险的。

必须维护副元素：

```text
C、P、S、N、Al、Cu、Ni、Cr、Mo 等
```

---

### 坑 2：把目标成分当等式

不要让模型强行：

```text
Mn = 1.20
```

而应使用：

```text
1.15 ≤ Mn ≤ 1.25
```

目标值用于后续偏差解释，不是硬约束。

---

### 坑 3：包装取整和 kg/t 混淆

必须明确：

```text
x_j = kg/t钢
kg_per_heat = x_j × 炉重
包数 = kg_per_heat / 包装规格
```

---

### 坑 4：Big-M 设置太大

`x_j ≤ M_j × y_j` 中的 `M_j` 不要无限大。

应该用业务上限：

```text
最大投料量
最大可能 kg/t
```

否则 MILP 求解性能和数值稳定性会变差。

# 第二阶段：鲁棒 MILP 与风险控制

## 2.1 第二阶段的目标

第二阶段的目标是：

> **让系统从“名义参数下最优”升级为“参数波动下仍然安全”。**

第一阶段默认：

```text
终点成分准确
合金成分准确
回收率准确
```

但现场并不是这样。

实际现场会有：

```text
终点 C 检测误差
终点 Mn 波动
合金批次成分波动
回收率波动
称量误差
钢水量误差
```

第二阶段重点处理前三个：

```text
1. 终点成分波动范围
2. 合金成分波动范围
3. 回收率波动范围
```

然后形成：

```text
鲁棒等级
最差情况成分校核
安全距离计算
鲁棒推荐方案
```

---

## 2.2 为什么第二阶段很关键？

第一阶段的模型会天然追求成本最低。

对于贵元素，它会贴下限：

```text
V 刚好高于下限
Nb 刚好高于下限
Mo 刚好高于下限
Ni 刚好高于下限
```

对于便宜但有风险的合金，它会贴上限：

```text
高碳锰铁便宜，但会把 C 顶高
硅锰便宜，但可能把 Si 顶高
高碳铬铁便宜，但会带 C
```

这会导致：

```text
理论上合格
现场上危险
```

第二阶段就是要解决：

```text
不要只在纸面上合格，
而要在合理波动下仍然合格。
```

---

## 2.3 第二阶段新增数据

第二阶段需要把第一阶段的单点参数扩展为：

```text
名义值
低值
高值
```

---

### 2.3.1 终点/残余成分波动范围

表结构从：

```text
元素 = 名义值
```

升级为：

| 元素 | 名义值 | 低值 | 高值 | 单位 |
|---|---:|---:|---:|---|
| C | 0.060 | 0.055 | 0.065 | wt% |
| Mn | 0.120 | 0.110 | 0.130 | wt% |
| Si | 0.010 | 0.005 | 0.015 | wt% |
| Cr | 0.030 | 0.025 | 0.035 | wt% |

含义：

```text
名义值：正常计算使用
低值：用于下限风险校核
高值：用于上限风险校核
```

例如：

```text
C 上限风险：用 C 高值
Mn 下限风险：用 Mn 低值
```

---

### 2.3.2 合金成分波动范围

表结构从：

```text
合金元素含量 = 名义值
```

升级为：

| 合金 | 元素 | 名义含量 | 低含量 | 高含量 | 单位 |
|---|---|---:|---:|---:|---|
| 硅锰 | Mn | 65 | 63 | 67 | % |
| 硅锰 | Si | 17 | 16 | 18 | % |
| 硅锰 | C | 1.8 | 1.5 | 2.2 | % |
| 高碳锰铁 | Mn | 75 | 73 | 77 | % |
| 高碳锰铁 | C | 7.0 | 6.5 | 7.5 | % |

对下限元素：

```text
用低含量
```

例如 Mn 要保证不低于下限，就用：

```text
硅锰 Mn 低含量
高碳锰铁 Mn 低含量
```

对上限元素：

```text
用高含量
```

例如 C 要保证不超过上限，就用：

```text
硅锰 C 高含量
高碳锰铁 C 高含量
```

---

### 2.3.3 回收率波动范围

回收率也从单值升级为三值：

| 合金 | 元素 | 名义回收率 | 低回收率 | 高回收率 |
|---|---|---:|---:|---:|
| 硅锰 | Mn | 0.95 | 0.92 | 0.97 |
| 硅锰 | Si | 0.90 | 0.86 | 0.93 |
| 高碳锰铁 | Mn | 0.96 | 0.93 | 0.98 |
| 高碳锰铁 | C | 0.95 | 0.93 | 0.98 |
| 钒铁 | V | 0.98 | 0.96 | 0.99 |

对下限元素：

```text
用低回收率
```

对上限元素：

```text
用高回收率
```

---

## 2.4 鲁棒等级设计

不要让现场用户直接面对复杂数学参数。

建议产品上设计成四个鲁棒等级：

| 鲁棒等级 | 名称 | 含义 | 适用场景 |
|---|---|---|---|
| R0 | 名义优化 | 只用名义值 | 理论测算、成本基线 |
| R1 | 轻度稳健 | 考虑部分波动 | 常规钢种 |
| R2 | 推荐稳健 | 中等保守 | 默认生产推荐 |
| R3 | 强稳健 | 按较不利情况计算 | 窄成分、高价值、投诉风险钢种 |

---

### 2.4.1 鲁棒等级如何落到参数上？

可以用一个系数 `α`：

| 等级 | α |
|---|---:|
| R0 | 0 |
| R1 | 0.33 |
| R2 | 0.67 |
| R3 | 1.00 |

对于低值：

```text
effective_low = nominal - α × (nominal - low)
```

对于高值：

```text
effective_high = nominal + α × (high - nominal)
```

举例：

```text
Mn 回收率名义值 = 0.95
Mn 回收率低值 = 0.92
```

R2 的有效低值：

```text
0.95 - 0.67 × (0.95 - 0.92)
= 0.95 - 0.0201
= 0.9299
```

R3 的有效低值就是：

```text
0.92
```

这样产品上可以说：

```text
R0：不保守
R1：轻度保守
R2：推荐保守
R3：强保守
```

而不用让用户理解复杂鲁棒优化术语。

---

## 2.5 鲁棒成分约束怎么写？

第二阶段的核心是把普通成分约束改成鲁棒成分约束。

---

### 2.5.1 对下限约束：用最不利低值

例如 Mn 需要满足：

```text
Mn ≥ Mn_min
```

普通模型是：

```text
b_Mn_nominal + Σ(x_j × a_Mn,j_nominal × r_Mn,j_nominal / 10) ≥ Mn_min
```

鲁棒模型改成：

```text
b_Mn_low + Σ(x_j × a_Mn,j_low × r_Mn,j_low / 10) ≥ Mn_min + margin_Mn_low
```

意思是：

```text
即使终点 Mn 偏低
即使合金 Mn 含量偏低
即使 Mn 回收率偏低
最终 Mn 仍然不能低于下限
```

---

### 2.5.2 对上限约束：用最不利高值

例如 C 需要满足：

```text
C ≤ C_max
```

普通模型是：

```text
b_C_nominal + Σ(x_j × a_C,j_nominal × r_C,j_nominal / 10) ≤ C_max
```

鲁棒模型改成：

```text
b_C_high + Σ(x_j × a_C,j_high × r_C,j_high / 10) ≤ C_max - margin_C_high
```

意思是：

```text
即使终点 C 偏高
即使高碳合金 C 含量偏高
即使 C 回收率偏高
最终 C 仍然不能超过上限
```

---

### 2.5.3 有上下限的元素要两边都校核

例如 Mn 有上下限：

```text
Mn_min ≤ Mn ≤ Mn_max
```

则需要两个约束：

下限：

```text
Mn_worst_low ≥ Mn_min + lower_margin
```

上限：

```text
Mn_worst_high ≤ Mn_max - upper_margin
```

不要以为 Mn 只需要防低。某些钢种 Mn 过高也会影响性能或标准命中。

---

## 2.6 最差情况成分校核

第二阶段输出时，不能只输出一个预测值。

要输出三类成分：

```text
名义预测值
最差低值预测
最差高值预测
```

---

### 2.6.1 名义预测

```text
Final_i_nominal =
b_i_nominal + Σ(x_j × a_ij_nominal × r_ij_nominal / 10)
```

表示正常情况下预计成分。

---

### 2.6.2 最差低值预测

```text
Final_i_worst_low =
b_i_low + Σ(x_j × a_ij_low × r_ij_low / 10)
```

用于判断是否可能低于下限。

---

### 2.6.3 最差高值预测

```text
Final_i_worst_high =
b_i_high + Σ(x_j × a_ij_high × r_ij_high / 10)
```

用于判断是否可能超过上限。

---

## 2.7 安全距离计算

第二阶段要输出“安全距离”，否则用户看不出风险。

---

### 2.7.1 下限安全距离

```text
lower_safety_distance_i = Final_i_worst_low - L_i
```

如果：

```text
lower_safety_distance_i > 0
```

说明最差情况下仍高于下限。

如果：

```text
lower_safety_distance_i < 0
```

说明最差情况下可能低于下限。

---

### 2.7.2 上限安全距离

```text
upper_safety_distance_i = U_i - Final_i_worst_high
```

如果：

```text
upper_safety_distance_i > 0
```

说明最差情况下仍低于上限。

如果：

```text
upper_safety_distance_i < 0
```

说明最差情况下可能超过上限。

---

### 2.7.3 综合安全距离

对每个元素取：

```text
safety_i = min(lower_safety_distance_i, upper_safety_distance_i)
```

对于只有下限的元素，只算下限距离。

对于只有上限的元素，只算上限距离。

---

## 2.8 风险等级划分

可以设置规则：

| 安全距离 | 风险等级 |
|---:|---|
| < 0 | 红色，高风险，最差情况下越界 |
| 0 ~ 0.002% | 橙色，贴边 |
| 0.002% ~ 0.005% | 黄色，中风险 |
| > 0.005% | 绿色，低风险 |

注意：不同元素阈值应该不同。

例如：

| 元素 | 风险阈值应该更敏感吗？ |
|---|---|
| B | 是，微量元素，极敏感 |
| C | 是，性能影响大 |
| Nb/V/Ti | 是，成本高且性能敏感 |
| Mn | 中等 |
| Si | 中等 |
| P/S | 是，上限必须严格 |

所以产品里应该支持：

```text
按元素配置风险阈值
```

| 元素 | 名义预测 | 最差低值 | 最差高值 | 内控范围 | 安全距离 | 风险 |
|---|---:|---:|---:|---:|---:|---|
| C | 0.162 | 0.154 | 0.175 | 0.150~0.180 | 0.005 | 低 |
| Si | 0.222 | 0.190 | 0.248 | 0.180~0.260 | 0.010 | 低 |
| Mn | 1.205 | 1.163 | 1.238 | 1.150~1.250 | 0.013 | 低 |
| V | 0.036 | 0.032 | 0.039 | 0.030~0.050 | 0.002 | 中 |

---

### 2.9.3 鲁棒性成本解释

| 成本项 | 元/t |
|---|---:|
| 理论 LP 成本 | 82.30 |
| 可执行 MILP 成本 | 83.10 |
| 鲁棒 MILP 成本 | 84.20 |
| 包装/投料规则带来的成本 | +0.80 |
| 鲁棒安全裕量带来的成本 | +1.10 |

这非常重要。

用户看到鲁棒方案比最低成本贵，会问：

```text
为什么多花钱？
```

系统要回答：

```text
多花的 1.10 元/t 是为了在终点成分、合金成分、回收率波动时，仍保证 C、Mn、V 等元素不越界。
```

---

## 2.10 第二阶段验收标准

| 验收项 | 要求 |
|---|---|
| 支持鲁棒等级 | R0/R1/R2/R3 可切换 |
| 支持参数上下限 | 终点、合金成分、回收率都能维护上下限 |
| 支持最差校核 | 输出 worst_low / worst_high |
| 支持安全距离 | 每个元素显示距离上下限的安全距离 |
| 支持风险等级 | 红黄绿风险可配置 |
| 成本可解释 | 鲁棒性成本能单独展示 |
| 方案合理 | 鲁棒方案不应极端保守，也不应贴边 |
| 参数缺失可处理 | 某元素没有上下限时有默认策略 |

---

## 2.11 第二阶段容易踩的坑

### 坑 1：所有参数都取最差，导致过度保守

如果所有元素都按最差情况，可能成本显著升高。

所以要用鲁棒等级：

```text
R0/R1/R2/R3
```

而不是只有“开/关”。

---

### 坑 2：安全裕量和鲁棒范围重复计算

如果你已经用了：

```text
终点低值
合金低值
回收率低值
```

又加很大的 margin，可能过度保守。

建议：

```text
鲁棒范围 = 参数不确定性
安全裕量 = 管理偏好或客户风险要求
```

两者要分开配置。

---

### 坑 3：所有元素用同一风险阈值

B 和 Mn 不能用同一个风险判断尺度。

应该按元素配置。

---

### 坑 4：只输出鲁棒方案，不输出成本差异

用户会觉得系统“算贵了”。

必须展示：

```text
可执行最低成本
鲁棒推荐成本
鲁棒性成本
风险降低效果
```

---

# 第三阶段：诊断、解释、替代、敏感性、复盘

## 3.1 第三阶段的目标

第三阶段是系统真正从“优化工具”升级成“决策系统”的关键。

第三阶段要解决：

```text
1. 为什么无解？
2. 为什么推荐这个方案？
3. 如果某个合金不用，怎么办？
4. 哪个价格变化最影响成本？
5. 哪个元素约束最卡成本？
6. 历史上系统算得准不准？
7. 哪些回收率和安全裕量需要修正？
```

第三阶段新增五大能力：

```text
1. 无解诊断
2. 方案解释
3. 替代合金分析
4. 成本敏感性分析
5. 历史方案复盘
```

---

# 3.2 能力一：无解诊断

## 3.2.1 为什么无解诊断很重要？

现场使用优化系统时，经常会遇到无解。

例如：

```text
终点 C 太高
目标 Mn 又很高
可用的锰系合金都带 C
低碳锰铁被禁用或价格缺失
```

系统可能求不出方案。

如果系统只提示：

```text
无可行解
```

现场不会接受。

必须告诉用户：

```text
为什么无解？
哪个约束冲突？
需要放松哪个条件？
是数据问题，还是工艺条件真的不满足？
```

---

## 3.2.2 无解类型分类

第三阶段建议把无解分成 6 类。

| 类型 | 说明 | 示例 |
|---|---|---|
| 数据缺失型 | 必要数据没维护 | Nb 有下限，但没有含 Nb 合金 |
| 初始成分越界型 | 终点/残余本身已超上限 | 终点 C 已高于 C 上限 |
| 成分冲突型 | 多元素约束互相矛盾 | Mn 要补，但补 Mn 会导致 C 超 |
| 投料规则冲突型 | 禁用、最大投料、最小投料冲突 | 唯一可补 V 的钒铁被禁用 |
| 包装取整冲突型 | 连续可行，整数袋数不可行 | B 范围太窄，包装太大 |
| 鲁棒过严型 | 名义可行，鲁棒不可行 | R3 太保守导致无解 |

---

## 3.2.3 无解诊断方法

建议用三层诊断。

---

### 第一层：求解前数据校验

在调用求解器之前先检查。

例如：

```text
元素有下限，但没有任何可补该元素的合金；
合金价格为空；
回收率为空；
内控下限 > 内控上限；
终点成分已超过上限；
包装规格 ≤ 0；
合金成分超过 100%；
回收率不在 0~1。
```

这类问题不要交给求解器，直接提示数据错误。

---

### 第二层：分层求解诊断

按模型复杂度逐步求解：

```text
1. 只考虑成分约束，连续 LP 是否可行？
2. 加入禁用规则，是否可行？
3. 加入最大/最小投料量，是否可行？
4. 加入包装取整，是否可行？
5. 加入鲁棒约束，是否可行？
```

通过这种方式可以定位：

```text
到底是哪一层把模型搞无解了。
```

例如：

| 求解层级 | 结果 |
|---|---|
| 连续 LP | 可行 |
| 加禁用规则 | 可行 |
| 加最小投料量 | 可行 |
| 加包装取整 | 不可行 |
| 加鲁棒约束 | 未执行 |

结论：

```text
无解原因主要来自包装取整，不是成分约束本身。
```

---

### 第三层：松弛变量诊断

给每个约束加松弛变量。

例如原来 C 上限：

```text
C_final ≤ C_max
```

改成：

```text
C_final ≤ C_max + slack_C_upper
```

并让模型最小化松弛量：

```text
Minimize 大惩罚 × Σ slack
```

如果最终：

```text
slack_C_upper = 0.006%
```

说明：

```text
只要 C 上限放宽 0.006%，模型就可行。
```

这可以转成业务解释：

```text
当前参数下 C 上限至少需要放宽 0.006%，或者必须减少高碳合金、启用低碳合金。
```

---

## 3.2.4 无解诊断输出示例

```text
当前模型无可行解。

主要冲突：
1. C 上限约束与 Mn 下限约束冲突。
2. 当前终点 C = 0.172%，内控 C 上限 = 0.180%，剩余增碳空间仅 0.008%。
3. 为满足 Mn 下限，当前可用锰系合金最少会带入 C = 0.014%。
4. 因此最小预测 C = 0.186%，超过内控上限 0.006%。

建议处理：
1. 启用低碳锰铁或金属锰；
2. 降低高碳锰铁最大使用量；
3. 放宽 C 内控上限；
4. 重新确认终点 C；
5. 降低鲁棒等级。
```

---

# 3.3 能力二：方案解释

## 3.3.1 为什么需要方案解释？

优化模型算出的方案可能是对的，但现场会问：

```text
为什么不用高碳锰铁？
为什么低碳锰铁这么贵还要用？
为什么 V 只加到下限附近？
为什么成本比 Excel 高？
为什么硅锰用量变少？
```

如果解释不了，现场会认为系统是黑箱。

方案解释就是把数学结果翻译成业务语言。

---

## 3.3.2 方案解释内容

至少解释 6 件事。

---

### 解释 1：成本为什么是这个数？

输出：

| 项目 | 数值 |
|---|---:|
| 合金总成本 | 84.20 元/t |
| 合金总单耗 | 12.45 kg/t |
| 最贵合金 | 钒铁 |
| 成本占比最高合金 | 硅锰 |
| 相比 Excel 方案 | -1.30 元/t |
| 相比最低成本方案 | +0.60 元/t |

文字解释：

```text
本方案总成本为 84.20 元/t。
其中硅锰贡献 24.80 元/t，钒铁贡献 18.00 元/t。
相比原 Excel 方案降低 1.30 元/t。
相比最低成本方案增加 0.60 元/t，主要用于降低 C 上限风险。
```

---

### 解释 2：每个合金的作用是什么？

| 合金 | 主要作用 | 附带影响 |
|---|---|---|
| 硅锰 | 补 Mn、Si | 带入少量 C、P、S |
| 高碳锰铁 | 低成本补 Mn | 明显带入 C |
| 低碳锰铁 | 在 C 约束下补 Mn | 成本较高 |
| 硅铁 | 补 Si | 对 C 影响小 |
| 钒铁 | 补 V | 成本高 |

---

### 解释 3：每个元素从哪里来？

以 Mn 为例：

| 来源 | Mn 贡献 % |
|---|---:|
| 终点 Mn | 0.120 |
| 硅锰带入 | 0.650 |
| 高碳锰铁带入 | 0.300 |
| 低碳锰铁带入 | 0.135 |
| 最终预测 Mn | 1.205 |

以 C 为例：

| 来源 | C 贡献 % |
|---|---:|
| 终点 C | 0.060 |
| 硅锰带入 | 0.004 |
| 高碳锰铁带入 | 0.030 |
| 低碳锰铁带入 | 0.002 |
| 高碳铬铁带入 | 0.010 |
| 最终预测 C | 0.106 |

这样现场可以看清楚：

```text
哪个合金在补主元素；
哪个合金在把风险元素顶高。
```

---

### 解释 4：哪些约束限制了方案？

系统应该自动识别“接近触发”的约束。

例如：

| 约束 | 状态 | 解释 |
|---|---|---|
| C 上限 | 接近触发 | 限制了高碳锰铁继续增加 |
| Mn 下限 | 接近触发 | 必须保留足够锰系合金 |
| V 下限 | 接近触发 | 钒铁成本高，系统只保留安全裕量 |
| Si 上限 | 未触发 | 允许继续使用一定硅锰 |

自动生成说明：

```text
C 上限约束接近触发，因此系统没有继续增加高碳锰铁，而选择部分低碳锰铁补 Mn。
```

---

### 解释 5：为什么不用某个更便宜的合金？

例如：

```text
为什么不用更多高碳锰铁？
```

系统回答：

```text
高碳锰铁单位 Mn 成本较低，但会带入较多 C。
当前 C 最差上限预测已经接近内控上限，继续增加高碳锰铁会导致 C 超限。
因此系统选择了部分低碳锰铁替代。
```

---

### 解释 6：为什么推荐方案不是最低成本方案？

```text
最低成本方案成本为 83.60 元/t，但 C 最差上限距离内控上限仅 0.001%。
推荐方案成本为 84.20 元/t，成本增加 0.60 元/t，但 C 安全距离提高到 0.006%。
因此推荐方案更适合现场生产。
```

这就是“多方案解释”的核心价值。

---

# 3.4 能力三：替代合金分析

## 3.4.1 替代合金分析解决什么问题？

现场经常会遇到：

```text
某合金缺货
某合金价格突然上涨
某供应商批次质量异常
某合金临时禁用
某钢种不允许用某类合金
```

系统要能回答：

```text
如果不用这个合金，还能不能生产？
成本增加多少？
风险变化多少？
替代组合是什么？
```

---

## 3.4.2 替代分析的基本方法

本质是做场景求解。

例如原方案是：

```text
硅锰 + 高碳锰铁 + 低碳锰铁 + 硅铁
```

做替代分析时，可以设置：

```text
y_硅锰 = 0
```

然后重新求解。

得到“不使用硅锰”的最优方案。

---

## 3.4.3 常见替代场景

| 场景 | 模型处理 |
|---|---|
| 不使用硅锰 | `y_硅锰 = 0` |
| 不使用高碳锰铁 | `y_高碳锰铁 = 0` |
| 不使用低碳锰铁 | `y_低碳锰铁 = 0` |
| 不使用某供应商合金 | 禁用该供应商所有材料 |
| 限制高碳合金用量 | `x_高碳锰铁 ≤ 上限` |
| 尽量少用高价合金 | 加入成本或使用惩罚 |
| 限制合金种类 | `Σy_j ≤ K` |

---

## 3.4.4 替代分析输出

| 场景 | 是否可行 | 成本 元/t | 成本变化 | 风险变化 | 替代说明 |
|---|---|---:|---:|---|---|
| 原推荐方案 | 可行 | 84.20 | 0 | 低 | 默认推荐 |
| 不用硅锰 | 可行 | 87.10 | +2.90 | 中 | 用硅铁 + 锰铁替代 |
| 不用高碳锰铁 | 可行 | 85.60 | +1.40 | 低 | 低碳锰铁增加 |
| 不用低碳锰铁 | 不可行 | - | - | - | C 上限与 Mn 下限冲突 |
| 不用钒铁 | 不可行 | - | - | - | V 下限无法满足 |

---

## 3.4.5 替代分析的业务价值

它可以帮助采购和生产沟通：

```text
某合金涨价后，还值不值得用？
某合金缺货时，成本会增加多少？
某供应商质量不稳定，对生产影响多大？
哪种合金是关键瓶颈物料？
```

这已经不是单纯合金成本计算，而是合金供应策略分析。

---

# 3.5 能力四：成本敏感性分析

## 3.5.1 成本敏感性分析解决什么问题？

它回答：

```text
哪个合金价格最影响成本？
哪个元素约束最影响成本？
哪个回收率最影响成本？
哪个终点成分变化最影响方案？
```

这对管理层、采购、工艺都很有价值。

---

## 3.5.2 价格敏感性

### 分析方法

对每种合金价格做上下浮动：

```text
-10%、-5%、0、+5%、+10%
```

每次重新求解。

输出：

```text
成本变化
方案是否切换
关键替代点
```

---

### 输出示例

| 合金 | 价格变化 | 总成本变化 | 方案是否变化 | 说明 |
|---|---:|---:|---|---|
| 硅锰 | +5% | +0.80 元/t | 不变 | 硅锰仍有性价比 |
| 硅锰 | +10% | +1.20 元/t | 变化 | 部分改用硅铁+锰铁 |
| 高碳锰铁 | +10% | +0.45 元/t | 不变 | 仍受 Mn 成本优势影响 |
| 低碳锰铁 | +10% | +0.20 元/t | 不变 | 用量小 |
| 钒铁 | +5% | +0.90 元/t | 不变 | V 无替代料，直接传导 |

---

## 3.5.3 价格临界点分析

临界点分析比简单敏感性更高级。

它回答：

```text
硅锰涨到多少钱时，系统会改用硅铁 + 锰铁？
高碳锰铁涨到多少钱时，低碳锰铁更划算？
钒铁价格变化是否会影响方案？
```

输出示例：

```text
当硅锰价格超过 7100 元/t 时，
系统开始减少硅锰用量，
并增加硅铁和高碳锰铁组合。
```

这个对采购很有价值。

---

## 3.5.4 元素约束敏感性

分析某个元素约束收紧或放宽对成本的影响。

例如：

```text
Mn 下限从 1.150 提高到 1.160
C 上限从 0.180 降低到 0.175
V 下限从 0.030 提高到 0.035
```

输出：

| 变化 | 成本变化 | 原因 |
|---|---:|---|
| Mn 下限 +0.01% | +0.65 元/t | 需要增加锰系合金 |
| C 上限 -0.005% | +1.20 元/t | 高碳锰铁受限，低碳锰铁增加 |
| V 下限 +0.005% | +4.50 元/t | 钒铁价格高，成本显著上升 |
| Si 上限 -0.02% | +0.80 元/t | 硅锰受限，锰铁增加 |

这可以帮助工艺人员理解：

```text
哪个成分控制要求最贵？
哪个元素的内控范围过窄？
哪些钢种成本被某个元素约束卡住？
```

---

## 3.5.5 回收率敏感性

分析回收率变化对成本和风险的影响。

例如：

```text
Mn 回收率 -1%
Si 回收率 -2%
V 回收率 -1%
Ti 回收率 -3%
```

输出：

| 参数变化 | 成本变化 | 风险变化 |
|---|---:|---|
| Mn 回收率 -1% | +0.45 元/t | Mn 下限风险升高 |
| Si 回收率 -2% | +0.30 元/t | Si 下限风险升高 |
| V 回收率 -1% | +0.90 元/t | V 成本增加明显 |
| C 回收率 +2% | +0.60 元/t | C 上限风险升高 |

这对后续修正回收率很重要。

---

## 3.5.6 终点成分敏感性

尤其关注终点 C。

例如：

| 终点 C | 推荐方案成本 | 方案变化 |
|---:|---:|---|
| 0.050 | 83.20 | 可多用高碳锰铁 |
| 0.060 | 84.20 | 默认方案 |
| 0.070 | 85.50 | 低碳锰铁增加 |
| 0.080 | 88.00 | 高碳合金明显受限 |
| 0.090 | 无解 | C 空间不足 |

这可以清楚说明：

```text
终点 C 对后续合金成本影响很大。
```

# 3.6 能力五：历史方案复盘

## 3.6.1 为什么要做历史复盘？

即使你们暂时不接外部系统，也应该让用户手动导入历史数据。

否则系统永远停留在：

```text
理论优化
```

无法变成：

```text
越用越准
```

历史复盘的目标：

```text
比较系统推荐、现场实际、最终检验之间的差异。
```

---

## 3.6.2 历史复盘需要记录什么？

每次计算方案都要归档。

### 推荐方案记录

| 字段 | 说明 |
|---|---|

## 3.6.3 复盘指标

### 成分预测偏差


| 元素 | 预测 | 实际 | 偏差 |
|---|---:|---:|---:|

| Mn | 1.205 | 1.187 | -0.018 |
| Si | 0.222 | 0.215 | -0.007 |
| V | 0.036 | 0.034 | -0.002 |

如果长期发现：

```text
Mn 实际总是比预测低
```

说明：

```text
Mn 回收率可能维护偏高
```

### 成本偏差

```text
成本偏差 = 实际成本 - 推荐成本
```

原因可能是：

```text
现场没有按推荐投料
包装取整不同
合金价格变化
人工为了保险多加
```

### 命中率

```text
成分命中率 = 成分全部落在内控范围内的炉次 / 总炉次
```

可以按：

```text
钢种
元素
鲁棒等级
班组
工艺路线
```

统计。

### 推荐采纳率

```text
采纳率 = 现场实际方案与系统推荐方案接近的炉次 / 总炉次
```

如果采纳率低，要分析：

```text

是现场习惯问题？
是数据维护不准？
是操作规则没有建进去？
```

# 1. 主数据核心：以“炼钢牌号”为唯一核心

你说得对：

> **炼钢牌号和钢种是一对多关系。**

所以原方案里“钢种成分表”需要改名并改逻辑。

## 1.1 正确业务关系

应该是：

```text
成品钢种 A ┐
成品钢种 B ├──→ 炼钢牌号 X ──→ 唯一内控成分目标
成品钢种 C ┘
```

例如：

```text
SPHC-A
SPHC-B
某客户专用牌号
某热卷商品牌号
```

可能最终都对应同一个：

```text
炼钢牌号：L1234
```

那么合金成本优化时，应该按：

```text
炼钢牌号 L1234
```

去取：

```text
C、Si、Mn、P、S、V、Nb、Ti 等内控目标
终点成分
回收率
合金规则
```

而不是按成品钢种各自维护一套重复目标。

---

## 1.2 新的数据结构

### 表 1：炼钢牌号表

```text
smelting_grade
```

| 字段 | 含义 |

| smelting_grade_name | 炼钢牌号名称 |
| process_route | 工艺路线，例如 BOF-LF-CC |
| enabled | 是否启用 |
| remark | 备注 |

---

### 表 2：成品钢种 / 订单牌号映射表

```text
product_grade_mapping
```

| 字段 | 含义 |
|---|---|
| product_grade_code | 成品钢种、销售牌号、订单牌号 |
| smelting_grade_id | 对应炼钢牌号 |
| customer_code | 可选，客户 |
| standard_code | 可选，执行标准 |

这张表解决：

```text
销售输入的是成品钢种；

```

---

### 表 3：炼钢牌号内控成分目标表

一行表示一个炼钢牌号的一个元素控制规则。

| 字段 | 示例 | 含义 |

业务上你可以认为：

```text
一个炼钢牌号唯一对应一套当前有效内控目标。
```

但系统设计上仍建议加版本字段：

```text
valid_from
valid_to
active_flag
```

因为现场内控目标以后一定可能调整。  
系统要保证的是：

> **同一时刻，一个炼钢牌号只能有一套生效内控目标。**

# 2. 合金成分是否唯一？不要假设唯一

你的第二个问题非常重要。

**相同合金名称，不应假设成分唯一。**

原因是：

```text
同样叫“硅锰”，不同供应商成分可能不同；
同样叫“高碳锰铁”，不同批次 C、Mn、P、S 会变；
采购标准有上下限，实际质保书有实测值；
同一物料编码在不同时期可能价格和成分都变；
现场有时只知道默认成分，有时能拿到批次检验成分。
```

所以设计时不要把“合金名称 = 唯一成分”。

---

## 2.1 推荐的通用合金层级

建议分成四层：

```text
合金大类
    ↓
合金规格 / 物料编码
    ↓
供应商 / 批次
    ↓
成分版本
```

例如：

```text
锰系合金
    ↓
硅锰 FeMn65Si17
    ↓
供应商A 2026-06 批次
    ↓
Mn=65.2%, Si=17.1%, C=1.8%, P=0.18%, S=0.025%
```

---

## 2.2 合金主数据表

```text
alloy_material
```

| 字段 | 含义 |
|---|---|
| alloy_id | 合金物料 ID |
| alloy_name | 合金名称，例如硅锰 |
| alloy_family | 合金大类，例如锰系、硅系、钒系 |
| material_code | 物料编码 |
| specification | 规格，例如 FeMn65Si17 |
| supplier_id | 可选，供应商 |
| enabled | 是否启用 |
| default_package_kg | 默认包装规格 |

---

## 2.3 合金成分版本表

```text
alloy_composition_profile
```

| 字段 | 含义 |
|---|---|
| composition_profile_id | 成分版本 ID |
| alloy_id | 合金物料 ID |
| source_type | DEFAULT / SUPPLIER_CERT / LAB_ACTUAL / MANUAL |
| batch_no | 批次号，可选 |
| valid_from | 生效时间 |
| valid_to | 失效时间 |
| remark | 备注 |

---

## 2.4 合金元素成分明细表

```text
alloy_composition_element
```

| 字段 | 示例 | 含义 |
|---|---:|---|
| composition_profile_id | CP001 | 成分版本 |
| element | Mn | 元素 |
| nominal_value | 65.0 | 名义含量 |
| low_value | 63.0 | 低值 |
| high_value | 67.0 | 高值 |
| unit | % | 单位 |

例如硅锰：

| 合金 | 元素 | 名义 | 低值 | 高值 |
|---|---:|---:|---:|
| 硅锰 | Mn | 65.0 | 63.0 | 67.0 |
| 硅锰 | Si | 17.0 | 16.0 | 18.0 |
| 硅锰 | C | 1.8 | 1.5 | 2.2 |
| 硅锰 | P | 0.18 | 0.15 | 0.25 |
| 硅锰 | S | 0.03 | 0.02 | 0.04 |

---

## 2.5 为什么要这么设计？

因为这样可以兼容三种使用场景：

### 场景 A：没有批次数据

使用默认成分：

```text
DEFAULT
```

适合理论成本测算和销售快速评估。

---

### 场景 B：有供应商质保书

使用供应商质保书成分：

```text
SUPPLIER_CERT
```

适合更精细的测算。

---

### 场景 C：有现场实测批次成分

使用实验室实际成分：

```text
LAB_ACTUAL
```

适合生产投料优化。

---

## 2.6 优化变量应该绑定到什么层级？

第一版建议绑定到：

```text
合金物料 alloy_id
```

高级版可以绑定到：

```text
合金批次 batch_no
```

也就是说，第一版求：

```text
硅锰加多少？
高碳锰铁加多少？
```

高级版求：

```text
硅锰-A批次加多少？
硅锰-B批次加多少？
高碳锰铁-C批次加多少？
```

这样才能兼容未来库存批次优化。

# 3. “最小投料量”的含义需要纠正

你理解得对。

“某合金一旦使用就至少投一定量”这句话容易误解。它不是说：

```text
只要这个合金在候选列表里，就必须投。
```

正确意思是：

> **如果优化器选择使用这个合金，那么它的投料量不能小于一个现场可执行的最小量。**

---

## 3.1 三个概念必须分清

| 概念 | 含义 |
|---|---|
| 可用合金 | 这个合金允许被系统选择 |
| 选中合金 | 优化器最终决定使用这个合金 |
| 最小投料量 | 如果选中了，不能低于该数量 |

---

## 3.2 数学表达

定义：

```text
x_j = 合金 j 的投料量，kg/t
y_j = 是否使用合金 j，0 或 1
```

约束：

```text
x_j ≤ max_j × y_j
x_j ≥ min_j × y_j
```

含义：

### 如果不使用

```text
y_j = 0
```

则：

```text
x_j ≤ 0
x_j = 0
```

这个合金不会被投。

---

### 如果使用

```text
y_j = 1
```

则：

```text
min_j ≤ x_j ≤ max_j
```

它必须达到最小投料量。

---

## 3.3 这不会强迫系统使用三个合金

你举的例子是对的：

> 可能启用了 3 个候选合金，但实际用 2 个就能满足目标。

这时优化器可以只选择 2 个。

例如候选合金：

```text
硅锰
高碳锰铁
低碳锰铁
```

如果：

```text
硅锰 + 高碳锰铁
```

就能满足 C、Si、Mn 约束且成本最低，那么模型会让：

```text
y_硅锰 = 1
y_高碳锰铁 = 1
y_低碳锰铁 = 0
```

低碳锰铁不会被使用。

---

## 3.4 为什么还需要最小投料量？

因为没有这个约束，优化器可能算出：

```text
硼铁 0.003 kg/t
钛铁 0.008 kg/t
低碳锰铁 0.012 kg/t
```

理论上能算，但现场没法投。

所以最小投料量的作用是：

```text
防止出现现场不可执行的极小投料。
```

---

## 3.5 还需要一个“少料种”目标

即使有最小投料量，模型仍可能为了省一点钱使用很多种合金。

所以建议加一个目标或约束：

```text
Σ y_j ≤ K
```

例如：

```text
一套方案最多使用 6 种合金。
```

或者在目标函数中加轻微惩罚：

```text
总成本 + λ × 使用合金种类数
```

更推荐的工程做法是：

```text
先求最低成本；
再在成本不超过最低成本 + Δ 的范围内，求更少料种方案。
```

这样比较符合现场决策。

# 4. 炉重可选：默认不强绑定炉重是合理的

你说：

> 炉重可选，默认不严格和炉重关联。

这个判断是合理的。

因为你们当前系统主要是：

```text
吨钢理论合金成本测算
```

核心单位应该是：

```text
kg/t钢
元/t钢
```

不一定每次都要关联具体一炉多少吨。

---

## 4.1 推荐支持三种计算模式

### 模式 A：吨钢理论模式

不输入炉重。

输出：

```text
kg/t
元/t
预测成分
```

适用：

```text
成本测算
方案比较
销售估价
牌号效益分析
```

这是默认模式。

---

### 模式 B：参考炉重模式

输入一个参考炉重，例如：

```text
150 t
```

输出：

```text
kg/t
kg/炉
元/t
元/炉
```

适用：

```text
工艺人员理解方案
预估单炉投料
```

但不强制包装取整。

---

### 模式 C：生产投料模式

输入实际炉重，并启用：

```text
包装取整
最小投料量
整数包数
```

输出：

```text
kg/t
kg/炉
袋数
取整后成本
取整后成分校核
```

适用：

```text
现场投料建议
生产执行
```

---

## 4.2 模型中如何处理炉重？

默认不输入炉重时，只求：

```text
x_j = kg/t
```

如果输入炉重 `W`：

```text
kg_per_heat_j = x_j × W
```

如果启用包装取整：

```text
kg_per_heat_j = pack_j × n_j
```

也就是：

```text
x_j × W = pack_j × n_j
```

如果炉重不输入，就不要启用这个整数包装约束。

# 5. 目标成分需要支持多种类型

你这个第 5 条非常关键。

不能只把目标成分设计成：

```text
目标值 ± 安全余量
```

现场确实存在多种目标类型。

---

# 5.1 推荐目标类型设计

建议每个元素都维护一个：

```text
target_type
```

用于描述控制方式。

---

## 类型 1：范围控制 RANGE

适用于大多数元素。

```text
lower_limit ≤ Final_i ≤ upper_limit
```

例如：

```text
C: 0.140% ~ 0.180%
Mn: 1.150% ~ 1.250%
```

---

## 类型 2：目标值加减余量 TARGET_BAND

你说的第一种情况：

> 存在安全余量，要求在加减余量范围内。

例如：

```text
目标 Mn = 1.20%
允许余量 ±0.03%
```

则：

```text
1.17% ≤ Mn ≤ 1.23%
```

表里可以维护：

```text
target_value = 1.20
lower_tolerance = 0.03
upper_tolerance = 0.03
```

系统转成：

```text
lower_limit = target_value - lower_tolerance
upper_limit = target_value + upper_tolerance
```

---

## 类型 3：低于目标值，越低越好 UPPER_LOWER_BETTER

你说的第二种情况：

> 给定目标值，例如低 Si，要求低于目标值，越低越好。

这不能只做成普通上限约束。应该分两层。

### 第一层：硬约束

```text
Final_Si ≤ Si_target
```

例如：

```text
Si ≤ 0.030%
```

### 第二层：偏好目标

在满足成本和其他成分约束的前提下：

```text
Si 越低越好
```

但是这里有一个业务问题：

> 为了让 Si 更低，是否允许增加成本？

所以建议系统提供三种策略。

---

### 策略 A：成本优先

```text
先保证 Si ≤ 目标值；
然后成本最低。
```

这适合普通成本测算。

---

### 策略 B：低 Si 优先

```text
先让 Si 尽可能低；
再在低 Si 方案中选择成本最低。
```

这适合客户对低 Si 很敏感的订单。

---

### 策略 C：成本窗口内低 Si

这是我最推荐的工程方式。

先求最低成本：

```text
C*
```

然后允许成本略微增加：

```text
Cost ≤ C* + Δ
```

在这个窗口内让 Si 更低：

```text
Minimize Final_Si
```

例如：

```text
最低成本方案：82.0 元/t，Si = 0.028%
允许多花 0.5 元/t
在 Cost ≤ 82.5 元/t 内求最低 Si
得到 Si = 0.021%
```

这样业务上容易解释：

> 多花 0.5 元/t，可以把 Si 从 0.028% 降到 0.021%。

---

## 类型 4：只需低于上限 MAX_ONLY

适用于 P、S、N、Cu 等有害或残余元素。

```text
Final_i ≤ upper_limit
```

系统只要不超上限即可，不需要追求某个目标值。

---

## 类型 5：只需高于下限 MIN_ONLY

适用于某些微合金元素。

```text
Final_i ≥ lower_limit
```

例如：

```text
V ≥ 0.030%
Nb ≥ 0.020%
```

成本最低模型通常会把它们控制在下限附近，因为这些元素贵。

如果现场希望不要太贴边，可以加：

```text
lower_margin
```

---

## 类型 6：不控制，仅监控 MONITOR_ONLY

有些元素不参与优化约束，但需要显示预测值。

例如：

```text
Cr、Ni、Cu、Mo 残余元素
```

如果没有明确控制范围，可以先只展示：

```text
预测值
来源分解
```

但正式系统里最好也能维护上限，否则优化器可能钻空子。

---

# 6. 安全余量和第二阶段“成分波动范围”不是一回事

你问：

> 把每个元素都加上安全余量，是不是就和第二阶段各种成分波动范围相同了？

**不是一回事，但它们会共同作用。**

这点必须分清，否则系统会过度保守。

---

## 6.1 安全余量是什么？

安全余量是人为设定的控制策略。

例如内控范围是：

```text
C: 0.140% ~ 0.180%
```

为了不贴边，系统使用：

```text
C: 0.145% ~ 0.175%
```

这里：

```text
lower_margin = 0.005
upper_margin = 0.005
```

这叫：

```text
控制余量
```

它本质是：

> **我不希望方案贴着内控边界。**

---

## 6.2 成分波动范围是什么？

成分波动范围描述的是输入参数不确定性。

例如：

```text
终点 C 名义值 = 0.060%
终点 C 可能低到 0.055%
终点 C 可能高到 0.065%
```

或者：

```text
高碳锰铁 C 名义含量 = 7.0%
可能是 6.5% ~ 7.5%
```

或者：

```text
C 回收率名义值 = 0.95
可能是 0.93 ~ 0.98
```

这叫：

```text
参数不确定性
```

它本质是：

> **我知道输入参数不准，所以要考虑最差情况。**

---

## 6.3 两者怎么同时使用？

普通确定性约束：

```text
L_i ≤ Final_i ≤ U_i
```

加安全余量后：

```text
L_i + margin_low ≤ Final_i ≤ U_i - margin_high
```

加入鲁棒波动后：

```text
WorstLow_i ≥ L_i + margin_low
WorstHigh_i ≤ U_i - margin_high
```

也就是说：

```text
安全余量收缩目标范围；
鲁棒波动改变预测成分的计算方式。
```

---

## 6.4 举例

假设：

```text
Mn 内控下限 = 1.150%
安全余量 = 0.010%
```

那么系统希望：

```text
Mn 至少达到 1.160%
```

如果不考虑波动，用名义值算：

```text
Mn_nominal ≥ 1.160%
```

如果考虑波动，用最差低值算：

```text
Mn_worst_low ≥ 1.160%
```

第二种更保守。

---

## 6.5 防止重复保守

如果你已经设置了很大的安全余量，又把终点、合金成分、回收率全按最差值计算，方案可能会太贵。

所以建议系统把两类参数分开：

| 参数 | 含义 | 是否可选 |
|---|---|---|
| safety_margin | 管理控制余量 | 第一阶段就可以用 |
| uncertainty_range | 参数波动范围 | 第二阶段使用 |
| robust_level | 使用多少波动范围 | 第二阶段使用 |

产品上可以提供：

```text
成本优先：小余量 + R0/R1
均衡推荐：中等余量 + R2
稳健优先：较大余量 + R3
```

# 7. “下限元素用低回收率，上限元素用高回收率”是什么意思？

你没理解是正常的，这句话之前说得太简略。

它的准确说法应该是：

> 对于某个元素的**下限约束**，要用让该元素偏低的参数计算；  
> 对于某个元素的**上限约束**，要用让该元素偏高的参数计算。

不是说某个元素永远是“下限元素”或“上限元素”。

同一个元素可能同时有下限和上限。

---

## 7.1 回收率复习

回收率表示：

```text
合金中某元素实际进入钢水的比例
```

例如：

```text
加入 100 kg 高碳锰铁
其中 Mn = 75 kg
Mn 回收率 = 95%
实际进入钢水 Mn = 75 × 95% = 71.25 kg
```

---

## 7.2 对下限约束，为什么用低回收率？

假设 Mn 要满足：

```text
Mn ≥ 1.150%
```

风险是什么？

风险是：

```text
实际 Mn 不够，低于 1.150%
```

什么情况下 Mn 会不够？

```text
终点 Mn 偏低；
合金 Mn 含量偏低；
Mn 回收率偏低。
```

所以为了保证最差情况下也不低于下限，模型要用：

```text
终点 Mn 低值
合金 Mn 低含量
Mn 低回收率
```

公式：

```text
Mn_worst_low =
终点Mn_low + Σ(合金加入量 × 合金Mn_low × Mn回收率_low / 10)
```

要求：

```text
Mn_worst_low ≥ Mn_min
```

---

## 7.3 对上限约束，为什么用高回收率？

假设 C 要满足：

```text
C ≤ 0.180%
```

风险是什么？

风险是：

```text
实际 C 太高，超过 0.180%
```

什么情况下 C 会太高？

```text
终点 C 偏高；
合金 C 含量偏高；
C 回收率偏高。
```

所以模型要用：

```text
终点 C 高值
合金 C 高含量
C 高回收率
```

公式：

```text
C_worst_high =
终点C_high + Σ(合金加入量 × 合金C_high × C回收率_high / 10)
```

要求：

```text
C_worst_high ≤ C_max
```

---

## 7.4 同一个元素有上下限怎么办？

例如 Mn：

```text
1.150% ≤ Mn ≤ 1.250%
```

那要同时建两条鲁棒约束：

### 防止 Mn 太低

```text
Mn_worst_low ≥ 1.150%
```

用：

```text
终点Mn低值
合金Mn低含量
Mn低回收率
```

### 防止 Mn 太高

```text
Mn_worst_high ≤ 1.250%
```

用：

```text
终点Mn高值
合金Mn高含量
Mn高回收率
```

所以更准确的说法是：

```text
下限约束用低值；
上限约束用高值。
```

# 8. 增加宽厚分档下的吨钢单价统计分布

这个建议非常好，但它不是优化模型本体，而是**效益分析模型**。

```text
不同合金成本统计；
不同客户效益；
不同市场竞争力。
```

即使炼钢成分目标相同，最终效益也可能不同。

# 5. 目标成分类型设计

你补充的低 Si 场景很关键，所以目标类型不能只有一种。

## 5.1 支持的目标类型

| 类型 | 业务含义 | 数学表达 |
|---|---|---|
| RANGE | 普通范围控制 | `L ≤ element ≤ U` |
| TARGET_BAND | 目标值 ± 余量 | `target - tol_low ≤ element ≤ target + tol_high` |
| MAX_ONLY | 只控制上限 | `element ≤ U` |
| MIN_ONLY | 只控制下限 | `element ≥ L` |
| UPPER_LOWER_BETTER | 低于目标，越低越好 | `element ≤ U`，再优化越低越好 |

## 5.2 低 Si 的推荐处理方式

你选择的是：

> **允许多花一个成本窗口，例如最多 +1 元/t，窗口内 Si 越低越好，同时把多方案列出来让用户决策。**

### 第一步：成本优先方案

```text

    成本最低

得到最低成本 C*
```

### 第二步：成本窗口内低 Si 方案

```text

    Si ≤ Si_target
    Cost ≤ C* + 1 元/t
目标：
    Si 最低
```

### 第三步：列出多方案

| 方案 | 成本 | Si | 说明 |
|---|---:|---:|---|
| A 成本最低 | 82.30 元/t | 0.038% | 满足低 Si，但不追求更低 |


# 8. 最小投料量的准确解释

你的理解是对的。

“某合金一旦使用就至少投一定量”不是说系统必须使用它。

正确逻辑是：

```text
候选合金很多；
优化器可以选择其中一部分；
如果某合金被选择使用，则投料量不能小于现场可执行最小量。
```

数学表达：

```text
x_j = 合金 j 的投料量
y_j = 是否使用合金 j，0 或 1
```

约束：

```text
x_j ≤ max_j × y_j
x_j ≥ min_j × y_j
```

如果：

```text
y_j = 0
```

则：

```text
x_j = 0
```

如果：

```text
y_j = 1
```

则：

```text
min_j ≤ x_j ≤ max_j
```

所以如果两个合金就能满足目标，模型完全可以只选两个。

# 10. 安全余量和波动范围的关系

你问得很关键：

> 把每个元素都加上安全余量，是不是就和第二阶段各种成分波动范围相同？

答案：

```text
不是一回事。
```

---

## 10.1 安全余量是什么？

安全余量是人为控制策略。

例如内控范围：

```text
C：0.140 ~ 0.180
```

为了不贴边，系统实际用：

```text
C：0.145 ~ 0.175
```

这里：

```text
lower_margin = 0.005
upper_margin = 0.005
```

含义是：

```text
我不想让方案贴着边界。
```

---

## 10.2 波动范围是什么？

波动范围描述参数不确定性。

例如：

```text
终点 C 名义值 = 0.060
终点 C 可能是 0.055 ~ 0.065

高碳锰铁 C 名义值 = 7.0%
高碳锰铁 C 可能是 6.5% ~ 7.5%

C 回收率名义值 = 0.95
C 回收率可能是 0.93 ~ 0.98
```

含义是：

```text
输入参数本身不是绝对准确的。
```

---

## 10.3 两者如何共同使用？

普通约束：

```text
L ≤ Final ≤ U
```

加安全余量：

```text
L + margin_low ≤ Final ≤ U - margin_high
```

加鲁棒波动：

```text
WorstLow ≥ L + margin_low
WorstHigh ≤ U - margin_high
```

也就是说：

```text
安全余量收缩目标范围；
波动范围改变预测值计算方式。
```

二者可以同时存在，但不能无脑叠加太多，否则系统会过度保守、成本虚高。

# 11. 回收率“低值/高值”的解释

之前说：

```text
下限元素用低回收率；
上限元素用高回收率。
```

更准确说法是：

> **对下限约束，用会让元素偏低的参数；对上限约束，用会让元素偏高的参数。**

---

## 11.1 例如 Mn 下限

约束：

```text
Mn ≥ 1.150
```

风险是：

```text
实际 Mn 不够。
```

导致 Mn 不够的情况：

```text
终点 Mn 偏低
合金 Mn 含量偏低
Mn 回收率偏低
```

所以要校核：

```text
Mn_worst_low =
终点Mn_low + Σ(合金加入量 × 合金Mn_low × Mn回收率_low / 10)
```

要求：

```text
Mn_worst_low ≥ Mn_min
```

---

## 11.2 例如 C 上限

约束：

```text
C ≤ 0.180
```

风险是：

```text
实际 C 太高。
```

导致 C 太高的情况：

```text
终点 C 偏高
合金 C 含量偏高
C 回收率偏高
```

所以要校核：

```text
C_worst_high =
终点C_high + Σ(合金加入量 × 合金C_high × C回收率_high / 10)
```

要求：

```text
C_worst_high ≤ C_max
```

---

## 11.3 同一个元素有上下限时怎么办？

例如 Mn：

```text
1.150 ≤ Mn ≤ 1.250
```

要同时做两条校核：

```text
Mn_worst_low ≥ 1.150
Mn_worst_high ≤ 1.250
```

所以不是“Mn 永远用低回收率”，而是：

```text
校核下限时用低值；
校核上限时用高值。
```

# 20. 当前最需要确认的 4 个问题

为了继续细化成需求规格书，下面几个问题需要你们内部确认。

## 问题 1：低 Si 禁用硅锰/硅铁是否是硬规则？

也就是说：

```text
Si目标 <= 0.040 时，硅锰/硅铁必须禁用？
```

还是：

```text
优先禁用，但如果无解，允许提示用户启用替代方案？
```

建议设计成：

```text
默认硬禁用；
无解时允许生成“放宽规则方案”供用户查看。
```

---

## 问题 2：金属锰兜底是默认可用，还是无解后才启用？

建议：

```text
默认不进入常规候选；
当 Mn 缺口无法满足或低 Si/低 C 约束导致无解时，作为兜底方案启用。
```

这样更符合“兜底”含义。

---

## 问题 3：按组距修正具体修正哪些指标？

目前从截图看它是宽厚二维修正矩阵，但还不能确定具体作用对象。

需要确认它修正的是：

```text
A. 合金成本
B. 钢坯成本
C. 热卷成本
D. 销售价格
E. 吨钢效益
F. 电耗/煤气/成材率
G. 其它
```

系统设计可以先做成通用矩阵，后续配置作用指标。

---

## 问题 4：销售辅助报价是否需要显示“建议报价底线”？

如果只是工艺辅助，可以不显示底线。

如果面向销售，建议显示：


# 1. 规则层重新定义

你说得对：

```text
低 Si 禁用硅锰/硅铁
金属锰兜底
```

这些都不是通用规则，而是某些现场、某些订单、某些牌号下的特殊规则。

所以系统不能写死：

```text

```

而应该做成：

> **现场可维护的规则层。**

---

## 1.1 规则层的定位

规则层位于：

```text
基础数据
    ↓
规则层
    ↓
优化模型
    ↓
多方案输出
```

它的作用是：

> 根据现场、炼钢牌号、成品钢种、订单要求、元素目标、客户要求等条件，动态生成本次优化场景。

也就是决定：

```text
哪些合金可用
哪些合金禁用
哪些合金作为兜底
哪个元素采用低值优先
成本窗口是多少
鲁棒等级是多少
是否启用特殊费用
是否需要工艺确认
```

---

## 1.2 规则不建议一开始做成“代码插件”

虽然你说“插拔即用”，但第一版不建议直接让现场写 Python/JavaScript 脚本。

原因：

```text
1. 现场维护难度高
2. 容易写出冲突规则
3. 审计困难
4. 权限风险高
5. 出错后不好追责
6. 优化模型结果难解释
```

更推荐第一版做成：

```text
规则模板 + 条件表达式 + 动作配置
```

也就是低代码规则，不是任意代码。

# 3. 销售辅助报价：是否显示“建议报价底线”？

你问的核心是：

> 这个报价底线的基础是什么？如何计算？是否有参考性？

我的建议是：

```text
第一版不要直接叫“建议报价底线”。
建议先叫：
    成本参考线
    经营底线价
    建议报价区间
```

因为“报价底线”这个词太强，容易被销售理解成：

```text
低于这个价绝对不能卖
```

但实际上，它取决于成本模型、价格时效、利润策略、客户策略、市场行情，不能简单当成硬边界。

---

# 4. 报价底线的本质

报价底线不是市场价。

它本质上是：

> **在当前成本数据、历史成本分布、订单特殊要求、目标利润和风险准备条件下，系统计算出的内部成本红线或经营参考线。**

它解决的问题是：

```text
这个订单大概会不会亏？
特殊成分要求会增加多少成本？
这个宽厚规格历史上利润怎么样？
最低卖到多少大概率还能覆盖成本和目标利润？
```

但它不能解决：

```text
市场能接受什么价格？
客户愿意出多少钱？
竞品报价多少？
战略客户是否可以低毛利接单？
产能紧张时是否应该提高报价？
```

所以它是**辅助报价**，不是最终报价。

---

# 5. 报价底线至少应分三层

不要只给一个数字。

建议销售页面显示三条线：

```text
1. 成本保本线
2. 经营底线价
3. 建议报价区间
```

---

## 5.1 第一条：成本保本线

含义：

> 按当前成本模型估算，不考虑目标利润，只覆盖成本和必要风险准备的价格。

公式：

```text
成本保本线 = 完全成本 + 风险准备
```

如果是含税报价：

```text
含税成本保本线 = 不含税成本保本线 × 税率系数
```

例如图七里看到类似：

```text
热卷税率 hot_roll_tax_rate = 1.13
```

那就说明系统里税率应该参数化。实际用多少由现场配置。

---

## 5.2 第二条：经营底线价

含义：

> 覆盖完全成本，并满足最低利润要求的内部底线。

可以有两种算法。

---

### 算法 A：按吨钢目标利润

如果公司要求：

```text
最低吨钢利润 = 80 元/t
```

则：

```text
经营底线价 = 完全成本 + 风险准备 + 最低吨钢利润
```

含税：

```text
含税经营底线价 = 不含税经营底线价 × 税率系数
```

---

### 算法 B：按目标毛利率

如果公司要求：

```text
最低毛利率 = 5%
```

那么不能简单加 5%，而应使用：

```text
经营底线价 = (完全成本 + 风险准备) / (1 - 最低毛利率)
```

例如：

```text
完全成本 = 3500 元/t
风险准备 = 30 元/t
最低毛利率 = 5%
```

则：

```text
经营底线价 = 3530 / (1 - 0.05)
          = 3715.79 元/t
```

---

## 5.3 第三条：建议报价区间

含义：

> 结合历史同牌号、同宽厚分档、同客户或类似订单的成交/效益分布，给出建议报价范围。

例如：

```text
建议报价区间 = [经营底线价, 历史 P75 或市场参考价]
```

或者：

```text
建议报价区间下限 = max(经营底线价, 历史 P25 价格)
建议报价区间中位 = 历史 P50 价格
建议报价区间上限 = 历史 P75/P90 价格
```

前提是你们有历史销售价或历史效益数据。

如果没有销售价，只能给：

```text
成本参考区间
```

不能给真正报价区间。

# 6. 完全成本应该包括什么？

这是报价底线最关键的问题。

销售报价底线不能只基于合金成本。

合金成本只是炼钢成本的一部分。

完整逻辑应该是：

```text
合金成本
    ↓
钢坯成本
    ↓
热卷成本 / 成品成本
    ↓
完全成本
    ↓
经营底线价
    ↓
建议报价区间
```

---

## 6.1 合金成本

来自合金优化模型：

```text
合金成本 = Σ(合金单耗 kg/t × 合金价格 元/t合金 / 1000)
```

输出：

```text
元/t钢
```

---

## 6.2 钢坯成本

参考你图八里的结构，钢坯成本可以包括：

```text
材料成本
合金成本
炼钢加工成本
直接人工
固定费用
回收项
其它费用
```

注意：

> 如果“材料成本”已经包含合金成本，就不能再重复加一次合金成本。

所以系统必须给每个成本项加：

```text
cost_item_code
included_in_parent_flag
```

避免重复计算。

---

## 6.3 热卷成本

热卷成本可以包括：

```text
钢坯成本 × 钢坯单耗
轧钢电耗
轧钢煤气单耗
轧钢工艺件
期间费用
固定费用
认证费用
宽厚组距修正
```

你图七里看到类似参数：

```text
轧钢钢坯单耗
轧钢电单耗
轧钢高炉煤气单耗
期间费用
热卷税率
```

这些就是服务于热卷成本和效益测算。

---

## 6.4 完全成本

建议定义：

```text
完全成本 =
    热卷基础成本
  + 订单特殊成本
  + 客户特殊成本
  + 认证费用
  + 包装/运输/仓储等可选成本
  + 宽厚组距修正
```

如果只是厂内成本，不含物流，就不要加运输。  
如果销售报价需要到岸价或含运价，就必须加运输。

---

# 7. 报价底线的推荐计算公式

## 7.1 不含税成本保本线

```text
cost_floor_ex_tax =
    full_cost_ex_tax
  + risk_reserve
```

---

## 7.2 不含税经营底线价

如果用吨钢最低利润：

```text
business_floor_ex_tax =
    full_cost_ex_tax
  + risk_reserve
  + min_profit_yuan_per_ton
```

如果用最低毛利率：

```text
business_floor_ex_tax =
    (full_cost_ex_tax + risk_reserve)
    / (1 - min_gross_margin_rate)
```

---

## 7.3 含税经营底线价

如果税率参数是乘数形式，例如：

```text
tax_factor = 1.13
```

则：

```text
business_floor_tax_included =
    business_floor_ex_tax × tax_factor
```

如果税率参数是百分数，例如：

```text
vat_rate = 13%
```

则：

```text
business_floor_tax_included =
    business_floor_ex_tax × (1 + vat_rate)
```

---

## 7.4 考虑销售费用、返利、佣金

如果销售费用、返利、佣金是按照销售价格比例扣除，例如：

```text
rebate_rate = 2%
commission_rate = 1%
```

则底线价应写成：

```text
business_floor_ex_tax =
    (full_cost_ex_tax + risk_reserve + min_profit_yuan_per_ton)
    / (1 - rebate_rate - commission_rate)
```

否则会低估底线。

# 8. 风险准备怎么计算？

风险准备不是随便加一个数。建议支持三种方式。

---

## 8.1 方式 A：固定风险准备

例如：

```text
risk_reserve = 20 元/t
```

简单，但粗糙。

---

## 8.2 方式 B：鲁棒成本差额

利用你们第二阶段的鲁棒优化结果：

```text
risk_reserve =
    鲁棒方案成本 - 名义最低成本
```

例如：

```text
名义成本 = 82.3 元/t
鲁棒成本 = 84.1 元/t

合金风险准备 = 1.8 元/t
```

这种方式最适合你们的合金优化系统。

---

## 8.3 方式 C：历史分布分位数

按同炼钢牌号、同宽厚分档的历史成本分布：

```text
risk_reserve =
    P75成本 - 当前预测成本
```

或者稳健一点：

```text
risk_reserve =
    P90成本 - 当前预测成本
```

例如：

```text
当前预测热卷成本 = 3500 元/t
历史 P75 成本 = 3540 元/t

risk_reserve = 40 元/t
```

这适合第三阶段。

---

## 8.4 推荐做法

第一阶段：

```text
固定风险准备 / 不加风险准备
```

第二阶段：

```text
鲁棒成本差额
```

第三阶段：

```text
max(鲁棒成本差额, 历史 P75/P90 差额)
```

即：

```text
risk_reserve =
max(
    robust_cost - nominal_cost,
    historical_p75_cost - predicted_cost,
    configured_min_reserve
)
```

# 9. 报价底线是否有参考性？

```text

2. 内控目标正确
3. 合金价格新鲜
4. 成本参数新鲜
```

```text
销售价格缺失
正式制造成本不完整
```

## 9.3 建议增加“报价可信度等级”

# 16. 最终计算逻辑总结

销售辅助报价建议按这个链路算：

```text
订单信息
    ↓
成品钢种 + 宽厚 + 特殊要求
    ↓
映射炼钢牌号
    ↓
读取内控目标
    ↓
叠加订单特殊规则
    ↓
计算名义合金成本
    ↓
计算鲁棒合金成本
    ↓
计算钢坯/热卷完全成本
    ↓
叠加组距修正、客户修正、特殊费用
    ↓
得到成本保本线
    ↓
加目标利润或毛利率
    ↓
得到经营底线价
    ↓
结合历史同分档价格/效益分布
    ↓
输出建议报价区间和风险等级
```

一句话：

> **报价底线的基础不是合金成本，而是“完全成本 + 风险准备 + 最低经营利润”。**  
> 它有参考性，但必须显示数据来源、可信度、风险提示，并且不能替代正式报价审批。
